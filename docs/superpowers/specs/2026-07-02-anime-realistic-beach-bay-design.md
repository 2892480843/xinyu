# 心屿海湾真实动画感优化设计

Date: 2026-07-02

## Background

当前自由探索模式已经使用 React Three Fiber、Three.js、GLB 资产、toon 材质和 Sobel 墨线后期。海滩区域的模型数量充足，但截图中的夜海仍显得不真实，主要原因不是缺模型，而是海湾近景的渲染层次不足：

- 海面是大尺寸平面材质，缺少低频波动和分层高光，远看像硬板。
- 近岸浪花主要靠单个透明圆环和少量水体 GLB，缺少沿海岸推进的节奏。
- 浅滩、泡沫、海平线、雾和夜间星光没有形成同一套动画电影感。
- 海滩道具已经集中在 `getBeach()` 单一数据源，但近镜头下仍需要用水面和浪带把空间关系粘合起来。

本次设计优先解决用户截图中的海滩 / 夜海假感，不扩大到全探索模式的村落、农田、森林和 UI 重塑。

## Goals

1. 将心屿湾从“低模摆件 + 大海面平板”提升为“可游玩的动画场景”。
2. 保留现有 3D 玩法、碰撞、彩蛋、垂钓入口和海滩单一数据源。
3. 在高性能档启用动态水面与浪带，在低性能档保留稳定静态降级。
4. 用最小范围改动完成第一轮可见提升，避免全局视觉系统重构。

## Non-Goals

- 不重做所有 GLB 模型。
- 不引入真实物理水模拟。
- 不调整核心交互、任务进度、存档结构或 API。
- 不重写全探索模式的视觉体系。
- 不修改主导航、移动端主 UI 或后端逻辑。

## Recommended Approach

采用“海湾局部重做”方案：在 `ExploreMode.tsx` 中新增海湾专用动画水面和手绘浪带组件，替换当前最显眼的硬平面海水表现，并保留既有 GLB 海浪、泡沫、涟漪、鱼群、水母等细节。

### Why This Approach

| 方案 | 结论 | 原因 |
|---|---|---|
| 只调色和后期 | 不采用 | 无法修复海面硬板和浪花纸片的根因。 |
| 海湾局部重做 | 采用 | 直接处理截图问题，影响面小，验证清晰。 |
| 全探索风格统一 | 暂缓 | 范围大，容易触及村落、森林、农田和 UI 的联动风险。 |

## Visual Direction

目标风格为“真实动画感”：

- 保留漫画描边和 toon 色阶。
- 避免写实 PBR 水面带来的风格跳脱。
- 海水使用分层色带、轻微顶点波动、手绘高光和浅滩透明感。
- 浪带以动画分镜方式表达“海在动”，不追求真实流体。
- 夜间加强月光 / 星光反射和远雾，弱化硬直海平线。

术语说明：

| 术语 | 说明 |
|---|---|
| R3F | React Three Fiber，用 React 组件方式组织 Three.js 场景。 |
| Three.js | Web 端 3D 渲染库，负责相机、网格、材质、灯光和动画。 |
| GLB | 3D 模型文件格式，项目中的建筑、道具、角色多使用此格式。 |
| Toon 材质 | 卡通渲染材质，用少量色阶表现明暗，适合漫画 / 动画风格。 |
| Sobel 墨线 | 一种边缘检测后期效果，用来生成手绘轮廓线。 |

## Component Design

### `AnimatedAnimeSea`

替换当前海面大平面的材质表现，保持同样覆盖范围。

- 使用 `planeGeometry` 分段网格，在 `useFrame` 中做轻量顶点波动。
- 色彩基于 `visual.sea`、`visual.seaHighlight` 和环境时间推导。
- 高性能档启用动态波动；低性能档使用静态材质。
- 使用 `meshToonMaterial` 或轻量 shader，优先保持 toon 风格统一。

### `AnimeShoreBreaks`

新增 3-5 条沿海湾方向排列的手绘浪带。

- 位置从 `BAY_ANGLE`、`BAY_WATERLINE_RADIUS`、沿岸向量计算。
- 每条浪带使用弧形 / 椭圆几何，错开相位做缓慢推进和呼吸。
- 夜间降低纯白泡沫，改为蓝白、月光色和透明边缘。
- 低性能档减少数量和动画频率。

### `BayLightReflection`

增加浅滩反射和夜间星月高光。

- 在海湾浅水区叠加半透明椭圆片。
- 夜间或流星夜启用更明显的细长高光。
- 雨天降低反射锐度，避免与雨幕冲突。

### Beach Composition Adjustments

轻微调整海湾近景构图，不改变彩蛋逻辑。

- 棕榈和远处泡沫不压到画面边缘。
- 近岸泡沫与浅水区形成连续层次。
- 保留 `getBeach()` 作为海滩位置单一数据源。

## Data Flow

1. `ExploreScene` 根据 `visual` 和 `environment` 解析环境视觉参数。
2. `AnimatedAnimeSea` 接收 `visual`、`environment`、`tier` 和降级状态。
3. `AnimeShoreBreaks` 基于海湾常量生成浪带位置。
4. 现有 `Coastline` 继续负责 GLB 海岸模型和海滩道具。
5. `FishingWaterSensor`、彩蛋和碰撞逻辑继续使用原位置数据，不依赖新视觉层。

## Error Handling And Fallbacks

- 如果性能档为 `low`，水面保持静态或低顶点数。
- 如果运行时帧率触发 `degraded`，关闭动态浪带的高频动画。
- 新视觉层只读现有状态，不写入存档，不影响玩法状态恢复。
- 若 shader 或材质出现兼容问题，可回退到 `meshToonMaterial` + 顶点色方案。

## Testing Plan

| 验证项 | 方法 | 验收标准 |
|---|---|---|
| 单元测试 | `npm test` | 既有探索、海滩、垂钓相关测试通过。 |
| 构建 | `npm run build` | TypeScript 和 Vite 构建通过。 |
| 视觉检查 | Playwright 打开本地 Vite，进入探索模式并截图海湾夜景 | 海面不再像硬平板，浪带有层次，UI 不遮挡主体。 |
| 性能检查 | 高低性能档各跑一次截图或帧预算检查 | 低性能档不出现明显卡顿或黑屏。 |

## Implementation Boundaries

第一轮预计只修改：

- `frontend/src/components/ExploreMode.tsx`
- 必要时补充或调整 `frontend/src/lib/exploreEnvironment.ts`
- 必要时新增小范围测试，优先复用现有 `frontend/tests/*`

不修改：

- 后端服务
- 存档结构
- GLB 生成脚本
- 移动端 App 主流程

## Acceptance Criteria

- 海湾夜景截图中，海面具备可见的手绘动画层次，不再像单色硬板。
- 近岸浪花不再是孤立圆环，而是沿海岸连续推进的浪带。
- 海湾浅滩、泡沫和远海颜色在夜间形成统一动画感。
- 垂钓、海螺、鱼群、水母、海滩彩蛋仍能正常工作。
- 构建和相关测试通过，或明确记录未通过原因。

## Implementation Defaults

- 第一轮优先使用 `meshToonMaterial`、顶点色和轻量几何动画。只有当这种方式无法形成足够的水面层次时，再引入小型 shader。
- 如果高性能档动画水面开销偏大，优先降低几何分段和浪带数量，不扩大降级机制。
