# 心灵印记 PNG 票根预览设计

## 背景

当前 `frontend/src/components/NarrativeCard.tsx` 在心灵印记生成后展示一张玻璃卡片，并提供“保存为 PNG”按钮。点击后会直接用 `canvas.toBlob()` 生成 `xinyu-imprint.png` 并触发浏览器下载。这个流程能保存图片，但缺少下载前预览，也无法让用户确认即将保存的卡片质感。

用户已确认升级方向：保存 PNG 前增加一个预览窗口，并把卡片升级为 **“岛屿票根 / 收藏品票根”** 质感。重点是仪式感、收藏感和当前心屿界面的沉浸一致性。

## 目标

- 点击“保存为 PNG”后先打开预览窗口，不再立即下载。
- 预览窗口中展示最终会下载的 PNG 票根图。
- 票根卡片走收藏品方向：暗色岛屿纪念物、虚线副券区、编号、印章感、微光纹理。
- 用户可在预览窗口中取消或确认下载 PNG。
- 复用现有 `canvas` 生成能力，不引入新截图库或 UI 框架。
- 桌面端与移动端都能使用，窄屏下预览不溢出。
- 下载失败时给出明确状态提示，不让弹窗卡死。

## 非目标

- 不做图片编辑器，不支持用户手动改文字、颜色或模板。
- 不新增后端 API，不上传图片。
- 不做社交分享、复制图片到剪贴板、二维码或长按保存提示。
- 不改变叙事、情绪识别、记忆、岛屿成长等业务逻辑。
- 不重构 `NarrativeCard` 的朗读、对话和选择卡流程。

## 方案对比

| 方案 | 描述 | 优点 | 缺点 | 结论 |
|---|---|---|---|---|
| 直接下载后提示 | 保持现有逻辑，只优化生成图片样式 | 改动最小 | 没有预览窗口，不满足核心需求 | 放弃 |
| 预览弹窗 + 确认下载 | 先生成预览 URL，弹窗展示，用户确认后下载 | 满足预览、改动集中、体验清晰 | 需要管理对象 URL 生命周期 | 采用 |
| 完整模板编辑器 | 预览中支持切换模板、颜色、尺寸 | 可玩性强 | 超出本次需求，增加状态和测试成本 | 暂不做 |

## 交互设计

### 入口卡片

心灵印记区域保持当前位置和信息层级：

1. 标题仍为“心灵印记”。
2. 按钮文案从“保存为 PNG”保留不变，降低学习成本。
3. 点击按钮后进入“生成预览中”状态，按钮可禁用或显示轻量状态。
4. 生成成功后打开预览窗口。
5. 生成失败时在原卡片下方显示“预览生成失败，请稍后再试”。

### 预览窗口

预览窗口采用居中模态层：

- 背景压暗并加轻微模糊，保持对当前岛屿场景的空间感。
- 面板使用现有玻璃材质语言，宽度约 `min(92vw, 780px)`。
- 顶部显示“保存前预览”和简短状态，如“这张票根会保存为 PNG”。
- 主体展示图片预览，保持 16:9 横向比例。
- 底部提供两个按钮：`取消` 与 `下载 PNG`。
- 支持点击遮罩、按 `Escape`、点击取消关闭。

### 下载流程

1. 点击“保存为 PNG”。
2. 组件调用生成函数，得到 `Blob`。
3. 用 `URL.createObjectURL(blob)` 得到预览地址。
4. 打开预览窗口并展示图片。
5. 点击“下载 PNG”时复用同一个 `Blob` URL 触发下载。
6. 下载动作完成后保留窗口，显示“已生成 PNG”，用户可自行关闭。
7. 关闭窗口时释放对象 URL，避免内存泄漏。

## PNG 视觉设计

PNG 尺寸继续使用 `960 x 540`，适合横向分享和桌面预览。导出时按 `window.devicePixelRatio` 放大画布，保证高分屏清晰。

票根画面结构：

| 区域 | 内容 |
|---|---|
| 背景 | 深海蓝灰到暖棕的暗色渐变，叠加细微噪点和柔光 |
| 主券区 | 左侧约 72% 宽，放标题、印记正文、情绪标签 |
| 副券区 | 右侧约 28% 宽，用虚线分隔，放编号、品牌、日期或情绪短标 |
| 票根边框 | 细线双层边框，四角和虚线区带轻微磨损/冲孔感 |
| 印章 | 半透明圆形“XINYU / 心屿”印章，作为收藏品标记 |
| 正文 | 使用中文衬线回退字体，保持可读和温柔 |
| 元信息 | `心屿 · {情绪标签}` 与收藏编号，如 `NO. 0702-XXXX` |

收藏编号不要求后端持久化，可由当前时间和情绪生成稳定显示。编号只用于视觉纪念，不作为业务 ID。

## 组件边界

本次主要修改 `frontend/src/components/NarrativeCard.tsx`：

| 单元 | 职责 |
|---|---|
| `drawWrappedText` | 继续负责 canvas 文本换行 |
| `createImprintTicketBlob` | 新增或抽出：绘制票根 PNG，并返回 `Promise<Blob>` |
| `saveImprintAsPng` | 改为生成预览，而不是直接下载 |
| `downloadPreviewPng` | 新增：确认后触发下载 |
| `ImprintPreviewModal` | 可作为组件内小函数或 JSX 块，承载预览窗口 |

为了保持改动范围小，优先在同一文件内实现小型辅助函数；只有当代码明显变长或复用需求出现时，才拆到独立文件。

## 状态设计

`NarrativeCard` 增加轻量状态：

| 状态 | 类型 | 作用 |
|---|---|---|
| `previewUrl` | `string \| null` | 当前预览图片地址 |
| `previewBlob` | `Blob \| null` | 下载时复用的 PNG 数据 |
| `previewOpen` | `boolean` | 控制预览窗口显示 |
| `savingPreview` | `boolean` | 避免重复点击生成 |
| `saveMessage` | `string \| null` | 继续用于成功/失败提示 |

关闭窗口或组件卸载时，如果存在 `previewUrl`，必须调用 `URL.revokeObjectURL(previewUrl)`。

## 错误处理

- `result.imprint` 为空时不显示入口，也不生成图片。
- `canvas.getContext("2d")` 失败时显示失败提示。
- `canvas.toBlob()` 返回空时显示失败提示。
- 生成中重复点击直接忽略，避免多个对象 URL 互相覆盖。
- 预览窗口关闭时停止持有 `Blob` 和对象 URL。

## 可访问性

- 预览窗口使用 `role="dialog"` 与 `aria-modal="true"`。
- 标题用 `aria-labelledby` 关联。
- 关闭按钮和下载按钮使用明确文案。
- 支持 `Escape` 关闭。
- 遮罩点击关闭，但点击面板内容不关闭。
- 按钮触摸目标不小于 44px。

## 验收标准

| 验收项 | 标准 |
|---|---|
| 预览入口 | 点击“保存为 PNG”先显示预览窗口，不直接下载 |
| 图片一致性 | 预览图和下载图来自同一个 `Blob` |
| 票根质感 | PNG 包含暗色票根、虚线副券区、编号、印章、正文和元信息 |
| 关闭行为 | 点击取消、遮罩或按 `Escape` 可关闭窗口 |
| 内存释放 | 关闭窗口或组件卸载时释放 `previewUrl` |
| 错误反馈 | 生成失败时展示可见错误提示 |
| 响应式 | 375px 宽视口下预览不横向溢出，按钮不重叠 |
| 构建 | `frontend` 下执行 `npm run build` 通过 |
| 回归 | 朗读、继续聊聊、再说一次、选择卡仍可正常显示和点击 |

## 风险与处理

| 风险 | 影响 | 处理 |
|---|---|---|
| canvas 绘制代码变长 | `NarrativeCard` 可读性下降 | 只抽取小函数，必要时后续再拆文件 |
| 长印记文字溢出 | PNG 文字被截断 | 控制最大行数和字号，过长时缩小字号或截断尾部 |
| iOS 下载行为差异 | Safari 可能打开图片而不是下载 | 本轮使用标准 `<a download>`，后续如有真实设备反馈再补长按保存提示 |
| 对象 URL 未释放 | 长时间使用可能占内存 | 关闭与卸载统一 revoke |

## 术语说明

| 术语 | 说明 |
|---|---|
| PNG | 一种无损图片格式，适合保存文字清晰的卡片图 |
| 预览窗口 | 下载前弹出的确认面板，让用户先看到将保存的图片 |
| Canvas | 浏览器原生绘图能力，这里用来把票根卡片绘制成图片 |
| Blob | 浏览器中的二进制数据对象，这里代表生成好的 PNG 图片数据 |
| Object URL | 临时本地地址，用来让 `<img>` 预览和 `<a>` 下载同一个 Blob |
| 模态层 | 临时覆盖页面的对话框，用户处理完当前任务后再返回原页面 |
