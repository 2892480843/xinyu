# 移动端按 Web 首页重排设计

## 背景

当前项目已有独立移动端入口 `frontend/mobile.html` 与 `frontend/src/mobile`。现有移动端采用三 Tab App 壳：`岛屿 / 足迹 / 我`，倾诉默认通过底部 Sheet 打开。

用户已明确调整方向：移动端不继续扩展三 Tab App 壳，而是**直接按照 Web 端首页的信息结构重新改布局**。因此本次设计目标是把移动端改成 Web 首页的窄屏版：同一品牌层级、同一主输入、同一主行动按钮、同一组次级入口，同时保留移动端必要的安全区、滚动和全屏覆盖层适配。

## 目标

- 保留独立移动端入口 `frontend/mobile.html` 和 `frontend/src/mobile`。
- 移动端主屏按桌面 Web 首页重排，而不是继续使用三 Tab 主导航。
- `MoodInput` 直接显示在主屏，用户打开移动端即可倾诉。
- “上岛走走”保持为输入下方的主行动按钮，并复用现有探索模式。
- “回望这些天 / 问问岛屿 / 登高望岛”等 Web 端次入口在移动端紧凑展示。
- 身份、心象地图、回访留言、错误提示、夜间守望等 Web 端核心信息在移动端保留入口或轻量展示。
- 呼吸、叙事、安全提示、静默贝壳、心境石、时光机、登高望岛、自由探索继续使用全屏覆盖层，适配 `100dvh` 与 safe area。
- 通过构建、现有测试和移动视口视觉检查验证。

## 非目标

- 不重写桌面端 `frontend/src/pages/Home.tsx`。
- 不合并桌面端与移动端入口。
- 不新增后端 API。
- 不改变情绪分析、记忆、叙事、TTS、音乐和 3D 探索的业务语义。
- 不引入新的 UI 框架或图标库。
- 不把 3D 重资源加入 PWA 预缓存。

## Web 端结构对齐

移动端主屏按 Web 首页的顺序重排：

| Web 端区域 | 移动端处理 |
|---|---|
| 品牌标题、月亮图标、副标题 | 顶部保留，尺寸收紧 |
| 身份卡、心象地图 | 标题下方紧凑展示；心象地图可作为按钮或轻量卡入口 |
| 回访信、低语、修正信、夜间守望 | 主输入上方展示一张轻卡，优先级保持与 Web 端一致 |
| `MoodInput` | 直接显示在主屏，不再默认放入底部 Sheet |
| “上岛走走” | 输入下方主 CTA，复用 `island-cta` 视觉语言 |
| 回望、问问岛屿、登高望岛 | 主 CTA 下方紧凑链接组，窄屏自动换行 |
| 底部声明 | 主内容下方保留，避开安全区 |

## 移动端布局

### 主屏

主屏使用单列滚动布局：

1. 顶部品牌区。
2. 身份与心象入口区。
3. 岛屿留言区。
4. 倾诉输入区。
5. “上岛走走”主按钮。
6. 次入口链接组。
7. 底部服务声明。

页面根容器允许纵向滚动，避免短屏设备出现内容被底部遮挡或压缩。整体仍保留固定岛屿背景、粒子、音乐控件和生长光效。

### 输入区

`MoodInput` 直接在主屏渲染，保持 Web 端的主工作流：

- 提交后进入 `loading`。
- 可取消加载。
- 根据结果进入 `safety`、`breathing` 或 `narrative`。
- 非语言入口继续支持“静默贝壳”和“心境石”。

移动端不再使用全局底部倾诉 Sheet 作为默认输入方式。若后续发现键盘遮挡，可只为输入区增加局部键盘避让，不恢复三 Tab 结构。

### 次入口

次入口不再分散到“足迹” Tab，而是在主屏底部紧凑展示：

| 入口 | 显示条件 | 行为 |
|---|---|---|
| 回望这些天 | `memories.length > 0` | 打开 `TimeMachine` |
| 问问岛屿 | `memories.length > 0` 且有身份 | 打开 `IslandAssistant` |
| 登高望岛 | 有记忆或物件 | 打开 `IslandMap` |
| 心象地图 | 有身份 | 打开 `MindMap` 移动覆盖层或紧凑视图 |

| `frontend/src/mobile/components/MobileBrand.tsx` | 可继续复用，必要时微调尺寸 |

| `frontend/tests/mobile_visual_check.mjs` | 移动视口检查主屏、输入、CTA、次入口、覆盖层 |

## 状态与数据流

状态流继续复用当前移动端 `useReflectFlow`：

1. `HomeMobile` 读取本地匿名身份。
2. 身份就绪后拉取记忆、岛屿状态、物件、回访留言、低语和修正信。
3. 主屏 `MoodInput` 提交到 `flow.submit()`。
4. `reflectStream()` 优先，失败回退 `reflect()`。
5. 结果进入安全提示、呼吸或叙事。
6. 叙事或非语言仪式完成后刷新记忆、物件和岛屿状态。

探索、时光机、心象地图、登高望岛等覆盖层继续由 `HomeMobile` 持有打开状态。

## 视觉规则

- 主屏最大宽度保持 `34rem` 左右，窄屏使用 `calc(1rem + env(safe-area-inset-* ))`。
- 页面使用 `min-height: 100dvh`，但允许 `overflow-y: auto`。
- `MoodInput` 与主 CTA 的距离保持紧凑，避免中段真空。
- 按钮触摸目标不小于 44px。
- 长文案和入口链接自动换行，不挤压到屏幕外。
- 背景音乐控件保留移动顶部样式，不能遮挡品牌和输入区。
- 不新增装饰性图片或外链视觉资产。

## 验收标准

| 验收项 | 标准 |
|---|---|
| 构建 | `frontend` 下执行 `npm run build` 通过 |
| 结构测试 | `npm run test` 中移动端相关测试通过 |
| 移动入口 | `/mobile.html` 可独立打开并加载 `src/mobile/main.tsx` |
| 主屏 | 品牌、身份/心象入口、留言、输入、主 CTA、次入口、声明均可见或可滚动抵达 |
| 倾诉 | 主屏输入可提交；加载可取消；错误可重试或收起 |
| 覆盖层 | 安全提示、呼吸、叙事、静默贝壳、心境石、时光机、登高望岛、探索均可打开并返回 |
| 小屏 | 375px 宽与短屏高度下无文字重叠、CTA 遮挡或横向滚动 |
| PWA | 继续使用移动 manifest 和移动 shell 离线回退，不破坏现有安装能力 |
| 桌面回归 | `index.html` 与 `Home.tsx` 不因移动布局改造发生功能性改变 |

## 风险与处理

| 风险 | 影响 | 处理 |
|---|---|---|
| 移除三 Tab 后入口变多 | 主屏可能拥挤 | 次入口使用紧凑链接组，按条件显示 |
| `MoodInput` 直接上屏导致键盘遮挡 | 输入体验受影响 | 保留 `useKeyboardInset`，必要时局部增加滚动定位 |
| 桌面组件窄屏溢出 | 心象地图或助手可能撑宽 | 给覆盖层和触发区加移动容器约束 |
| 旧测试与新结构冲突 | 测试失败但行为正确 | 更新测试断言到 Web 对齐布局 |
| 未使用组件残留 | 代码看起来冗余 | 本轮不做删除型重构；后续稳定后再清理 |

## 术语说明

| 术语 | 说明 |
|---|---|
| Web 同构布局 | 这里指移动端沿用 Web 首页的信息顺序和主要操作路径，而不是另做一套 App Tab 架构 |
| CTA | Call To Action，主行动按钮；这里主要指“上岛走走” |
| Safe area | 手机刘海、底部手势条等不可遮挡区域；通过 CSS 的 `env(safe-area-inset-*)` 适配 |
| 覆盖层 | 临时盖住主屏的全屏体验，例如叙事、安全提示、时光机和自由探索 |
| PWA | Progressive Web App，渐进式 Web 应用；网页可安装到手机桌面，并具备一定离线能力 |
