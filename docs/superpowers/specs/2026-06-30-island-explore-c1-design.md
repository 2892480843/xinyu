# 上岛走走 C1 全岛十区与日内天气设计

## 背景

当前项目是 Vite + React + Three.js 的前端应用，`frontend/src/components/ExploreMode.tsx` 承担「上岛走走」自由探索体验。现有探索岛已经具备可走 3D 地形、环岛公路、中心村落、海湾沙滩、池塘、灯塔、码头、山庄、街区、全岛地图、迷你地图、位置环境音、夜晚切换、驾驶覆盖层和多种彩蛋。

用户已明确选择 C1：在同一座连续 3D 岛上全铺开新增地点，不拆成多个传送地图。此次设计目标是把现有探索岛扩展为十个清晰可逛小场景，并新增日出、中午、夕阳、夜晚和雨天状态。

## 目标

- 在现有「上岛走走」同一张 3D 岛地图中新增并强化十个区域：家、海滩、稻田、山、森林、小镇、农村、动物园、沼泽地、风景区。
- 每个区域都要具备地貌、可见模型、地图标注、环境音或氛围变化，以及至少一个轻互动或靠近提示。
- 将当前白天/夜晚布尔切换升级为时辰与天气系统：日出、中午、夕阳、夜晚、雨天。
- 保留现有心愿、NPC、精灵、种花、天灯、垂钓、风铃、开车、全岛地图等玩法。
- 通过集中数据源驱动 3D 场景、地图、位置音景和 UI，避免地图标签与实际场景脱节。
- 控制首屏进岛性能：新增完整小场景分批挂载，低性能设备使用较少实例和更轻的粒子效果。

## 非目标

- 不重写 `ExploreMode.tsx` 为全新游戏引擎。
- 不新增后端 API，不改变记忆、叙事、情绪分析或岛屿状态接口。
- 不引入新的大型第三方图形库。
- 不强制生成全新 GLB 资产；第一版优先复用 `frontend/public/models` 中已有房屋、自然、海滩、动物和地形模型。
- 不移除现有“去林间土路”驾驶地图；它仍作为已有额外兜风入口保留。

## 方案选择

| 方案 | 核心思路 | 优点 | 风险/成本 | 结论 |
|---|---|---|---|---|
| C1 同一座岛十区全铺开 | 所有地点放入现有 3D 岛，走路/开车/地图/音景连续 | 最符合“上岛走走”的连续沉浸感；复用当前地形、地图和音景基础 | 场景对象增多，需要分批挂载和低配降级 | 采用 |
| C2 主岛 + 开车切子地图 | 乡村、动物园、沼泽等独立地图 | 每张地图能做得更大 | 切图打断散步体验，实现面更广 | 不采用 |
| C3 分章节解锁 | 区域随进度逐步开放 | 节奏更游戏化 | 用户本轮希望使用 C1，一进岛就能看到更多地图 | 不采用 |

## 区域设计

| 区域 | 建议位置 | 视觉与模型 | 轻互动 | 音景/天气感 |
|---|---|---|---|---|
| 家 | 中心村落西南侧，靠近现有村落设施 | 复用 `houseCottage`、`houseLoft`、`isleWell`、`townMailbox`，形成小院、信箱、晾晒或长椅 | 靠近显示“回家坐一会儿”，可触发安静提示 | 白天鸟声，夜晚窗灯更暖，雨天屋檐雨声感 |
| 海滩 | 保留并强化东南海湾 | 复用棕榈、躺椅、冲浪板、沙堡、海螺、宝箱、海龟、寄居蟹 | 已有拾海、海螺、垂钓继续保留；补地图标签一致性 | 海浪、雨天浅滩更暗，夕阳反光增强 |
| 稻田 | 南部或东南内陆，避开海湾沙滩 | 复用 `xy_nat_crop_sprout.glb`，用实例化行列做田埂和水田 | 走近显示“稻田在风里轻轻摆”，可发现一只纸船或小灯 | 中午明亮，夕阳金黄，雨天水田泛光 |
| 山 | 西北高地与现有西岭山庄周边 | 复用岩石、石阶、山庄、松树、雪块；强化高地边界 | 到观景点触发“登高望岛”式提示 | 山风，日出和夕阳最明显 |
| 森林 | 西侧与北侧林地 | 复用松树、阔叶树、蘑菇、狐狸、猫头鹰 | 走近狐狸/蘑菇圈触发发现；雨天增加雾感 | 森林风声、夜晚虫鸣 |
| 小镇 | 中央偏南或现有老城街区 | 复用 `townblock`、街灯、招牌、长椅、售货机、咖啡屋 | 靠近邮筒、长椅或摊位触发短句 | 中午最热闹，夜晚街灯亮 |
| 农村 | 稻田旁或南部乡间 | 复用农舍、干草堆、木栅栏、风车、田间小路 | 靠近干草堆或风车触发乡间提示 | 鸟声、风车声感，雨天更安静 |
| 动物园 | 东侧内陆，和海湾/小镇保持距离 | 复用狐狸、猫、猫头鹰、鱼、海龟等小动物模型，增加低矮围栏和牌子 | 靠近动物提示名字或一句观察文案 | 白天轻快，雨天动物躲棚下 |
| 沼泽地 | 东南偏南低地，靠近浅水但不和海滩重叠 | 复用芦苇、莲花、蘑菇、浅水面、石头 | 走近莲叶或发光水洼触发“沼泽回声” | 雾、雨天、水声更突出 |
| 风景区 | 北部高处或山路入口 | 复用鸟居、神社、观景台、石灯、风铃 | 靠近观景台触发拍照式提示；可作为全岛俯瞰地标 | 日出/夕阳色彩最强，夜晚石灯亮 |

## 数据与架构

### 区域注册表

新增一个集中区域配置，供 3D、地图、音景和 UI 共用。建议先放在 `ExploreMode.tsx` 附近，若实现时体量过大再拆为 `frontend/src/lib/exploreZones.ts`。

| 字段 | 用途 |
|---|---|
| `key` | 区域稳定标识，如 `home`、`rice`、`swamp` |
| `label` | 地图与 UI 展示名称 |
| `x` / `z` | 世界坐标中心 |
| `radius` | 区域判定范围 |
| `kind` | 地图图标类型 |
| `color` | 地图标记色 |
| `ambience` | 对应位置音景 |
| `weatherAffinity` | 哪些天气/时辰下视觉增强 |

### 3D 场景组件

`ExploreScene` 内新增一个 `IslandDistricts` 层，按区域数据渲染完整小场景：

- `HomeDistrict`：家、小院、信箱、长椅。
- `RiceFieldDistrict`：稻田、水田、田埂、稻草人或纸船。
- `FarmDistrict`：农舍、干草堆、栅栏、风车周边。
- `ZooDistrict`：小动物围栏、牌子、可发现动物。
- `SwampDistrict`：浅水、芦苇、莲花、湿地石头。
- `ScenicDistrict`：观景台、鸟居、石灯、风铃。

现有 `Village`、`Town`、`Coastline`、森林彩蛋、山庄和灯塔继续保留，并逐步接入同一份区域配置。

### 地图系统

扩展 `MAP_POIS` 与 `PoiKind`：

- 新增 `home`、`rice`、`farm`、`mountain`、`forest`、`zoo`、`swamp`、`scenic`。
- 大地图展示全部十区标签。
- 小地图继续使用光点，不堆满文字。
- 图例从当前地标列表改为区域列表，避免底部标签过长时挤出屏幕。

### 位置音景

扩展 `LocationZone` 或新增区域到音景映射：

- 家/小镇/农村可复用 `meadow_day`、`meadow_night`。
- 稻田/农村白天优先鸟声，雨天叠加雨声。
- 沼泽地复用水声或新增湿地判定，第一版可用 `brook`。
- 森林/山继续使用 `wind_forest`。
- 海滩继续使用 `ocean_waves`。

## 日内与天气设计

### 状态模型

将当前 `night: boolean` 升级为探索专用环境状态：

| 状态 | 说明 |
|---|---|
| `timeOfDay` | `dawn`、`noon`、`sunset`、`night` |
| `weather` | `clear`、`rain` |

`night` 的本地存储可迁移为 `xy_explore_time`，雨天存储为 `xy_explore_weather`。读取旧 `xy_night=1` 时映射为 `timeOfDay="night"`，保持兼容。

### 视觉效果

| 状态 | 天空 | 光照 | 场景细节 |
|---|---|---|---|
| 日出 | 暖橙到浅蓝渐变 | 低角度暖光 | 风景区、山顶和海面更柔 |
| 中午 | 明亮蓝天 | 高强度白金光 | 稻田、农村、小镇最清晰 |
| 夕阳 | 橙红到紫色渐变 | 斜向暖光 | 海滩、观景台、风铃更突出 |
| 夜晚 | 深蓝紫夜空 | 低环境光 + 月亮/星光 | 现有星空、天灯、猫头鹰、蘑菇圈保留 |
| 雨天 | 灰蓝天空和雾 | 光照降低 | 雨幕、湿地/水田泛光、雨声音景 |

### UI 控制

左上菜单中将“切白天/切夜晚”替换为两个入口：

- `时辰`：日出 / 中午 / 夕阳 / 夜晚。
- `天气`：晴天 / 下雨。

移动端与桌面上岛探索共用该菜单，因为 `ExploreMode` 已被两端复用。

## 性能策略

- 新区域模型继续使用 `DelayedMount` 分批挂载，低性能设备延迟更长。
- 稻田、芦苇、栅栏、作物、动物园牌子等重复对象使用 `InstancedMesh` 或复用现有 `GltfProp` 小模型。
- 雨天效果使用轻量线段/点精灵，不使用昂贵全屏后期。
- 小动物和区域互动保持低频 proximity 检查，避免每帧 React `setState`。
- 大地图文字只在打开大图时展示，常驻迷你地图不展示十区文字。

## 验收标准

| 验收项 | 标准 |
|---|---|
| 十区可见 | 全岛地图能看到家、海滩、稻田、山、森林、小镇、农村、动物园、沼泽地、风景区 |
| 十区可走 | 玩家能在同一张 3D 岛上走到每个区域，不需要切换地图 |
| 十区有内容 | 每个区域至少有独立地貌/模型组合和一条靠近提示或发现反馈 |
| 时辰切换 | 日出、中午、夕阳、夜晚能从菜单切换，天空、光照、地图夜色和现有夜间玩法保持一致 |
| 雨天切换 | 晴天/下雨能从菜单切换；雨幕、雾感、雨声或雨天视觉反馈生效 |
| 兼容旧状态 | 已存 `xy_night` 的用户进入后仍能得到合理夜晚状态 |
| 性能 | 桌面端进入探索不出现长时间黑屏；低性能档不强制渲染高数量雨滴和实例 |
| 构建 | 在 `frontend` 下执行 `npm run build` 通过 |
| 自动测试 | 现有 `npm test` 通过，并新增覆盖区域注册、地图标签、时辰/天气状态的测试 |

## 风险与处理

| 风险 | 影响 | 处理 |
|---|---|---|
| `ExploreMode.tsx` 已接近 8k 行 | 继续膨胀会降低可维护性 | 区域数据和纯函数优先拆到 `lib`；场景组件保持小块命名 |
| 十区完整场景导致加载变慢 | 首次进岛卡顿或空白 | 继续使用 `Suspense`、`DelayedMount`、低配减少实例数量 |
| 地图标签过密 | 大地图文字重叠，小地图难读 | 小地图只显示光点，大地图使用标签偏移和图例折行 |
| 区域与现有地标穿模 | 新模型可能和道路、树林、海湾道具重叠 | 坐标先避开 `ROAD_CTRL_PTS`、`LANDMARK_PADS`、海滩判定；必要时加入避让半径 |
| 雨天与夜晚叠加过暗 | 用户看不清可走路径 | 雨天保留最低环境光和浅色 UI，夜雨降低雨滴密度 |
| 动物园动物素材有限 | “动物园”可能显得单薄 | 第一版复用狐狸、猫、猫头鹰、鱼、海龟，重点做围栏、牌子和靠近反馈 |

## 术语说明

| 术语 | 说明 |
|---|---|
| 3D 岛 | 玩家在浏览器中实时行走的 Three.js 场景，不是静态背景图 |
| Three.js | Web 3D 渲染库，用来绘制岛屿、模型、光照、天空和相机 |
| R3F | React Three Fiber，让 Three.js 场景以 React 组件方式组织 |
| GLB | 3D 模型文件格式，项目里的房屋、动物、树木、海滩道具多为 `.glb` |
| `Suspense` | React 的延迟加载边界，模型未加载完时不阻塞整个页面 |
| `DelayedMount` | 项目已有的延迟挂载组件，用来错峰加载重模型，降低进岛瞬间卡顿 |
| `InstancedMesh` | Three.js 的批量渲染方式，适合稻田、芦苇、栅栏等大量重复物件 |
| POI | Point of Interest，兴趣点；这里指地图上的区域或地标标记 |
| 音景 | 根据玩家位置切换的环境底噪，例如海浪、鸟声、风声、溪流声 |
| Proximity | 靠近判定；玩家走到某个范围内时触发提示、发现或交互 |
