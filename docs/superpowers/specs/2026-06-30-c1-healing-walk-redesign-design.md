# C1 Healing Walk Redesign Design

## Brief

用户反馈 C1 已有地图、时辰和下雨功能，但整体效果没有达到预期。本轮选择 B 方向：把“上岛走走”从偏俯视的全岛展示，调整为更贴地、更温柔、更容易识别地点的治愈散步体验。

## Design Read

- 当前功能面基本齐全，但第一眼仍偏“航拍看地图”，不是“走在岛上”。
- 体验目标不是继续堆新地点，而是让已有地点在近景里更有存在感。
- 视觉语言应靠近低密度、手作小岛、温柔近景、轻天气氛围。

## Dial Values

| Dial | Value | Reason |
|---|---:|---|
| Variance | 5 | 每个区域需要有不同剪影和地面质感，但保持同一个小岛世界观 |
| Motion | 4 | 雨、草、相机缓入要有气息，避免过度炫技 |
| Density | 3 | 保持可散步、可停留，不把画面塞满 |

## Goals

- 开场相机从低角度缓入，不再用高空俯瞰作为第一印象。
- 常规跟随相机保持近景散步感，角色和眼前地点更大、更可读。
- 家、稻田、农村、动物园、沼泽地、风景区等区域增加地面色块和更清楚的道具组合。
- 下雨天更明显：天空、雾、雨丝和地面湿润感共同变化。
- 保留 C1 已有地图、时辰、天气、区域提示和环境音逻辑。

## Non-Goals

- 不新增复杂任务系统。
- 不新增存档结构或接口契约。
- 不替换核心角色模型。
- 不做大规模资产生成，优先复用现有 GLB 与程序化几何。

## Acceptance

- 自动化测试能约束 healing walk 相机、区域呈现和雨天强度参数。
- `npm test` 和 `npm run build` 通过。
- 桌面截图能看到更近的相机、更清楚的区域地面与道具。
- 雨天截图中天空、雾、雨丝和湿润地面比 C1 第一版更明显。
