# 真实垂钓完整系统设计

## 背景

当前项目是 Vite + React + Three.js 的前端应用，`frontend/src/components/ExploreMode.tsx` 承担「上岛走走」自由探索体验。垂钓已有基础实现：玩家走到海湾岸边后可以触发垂钓，显示 `xy_item_fishing_bobber.glb` 浮漂，等待后进入咬钩状态，再通过节奏窗口收线并获得收获卡；规则辅助函数位于 `frontend/src/lib/fishing.ts`。

这套实现解决了「能钓」的问题，但仍像 HUD 小游戏，不像真实垂钓。用户明确要求「真实的垂钓」，并在方案选择中确认采用 C 档完整钓鱼系统：同时具备操作模拟、生态结果、装备饵料、图鉴记录，以及主角本人执行的钓鱼动作。

本设计基于当前本地代码确认以下事实：

- 当前默认可玩主角是 `hero`，模型路径为 `frontend/public/models/xyshz_rigged.glb`，在代码中通过 `MODELS.heroChar: "/models/xyshz_rigged.glb?v=6"` 接入。
- 现有角色动作通过 `frontend/src/lib/protagonistAction.ts` 的 `CharacterActionClip` 与 `selectCharacterAction` 路由到 `ExploreMode.tsx` 的 `Player`、`GltfHero`、`GltfGuardian` 和 `GltfPocoyo`。
- 垂钓浮漂已有专用 GLB：`frontend/public/models/xy_item_fishing_bobber.glb`。
- 当前垂钓计数使用 `localStorage.xy_catch`，暂未接后端持久化。

## 已确认决策

| 决策 | 结论 |
|---|---|
| 玩法深度 | 采用 C 档完整钓鱼系统 |
| 真实感来源 | 操作模拟 + 生态结果，而不是单纯扩展奖励文案 |
| 角色要求 | 必须是当前可玩主角本人垂钓，默认验收对象为 `hero` / `xyshz_rigged.glb?v=6` |
| 场景边界 | 仍挂在 `ExploreMode` 的海湾自由探索互动内，不拆独立页面 |
| 持久化 | 第一版使用版本化 `localStorage`，暂不新增后端 API |
| 规则边界 | 核心规则拆到 `frontend/src/lib`，避免继续膨胀 `ExploreMode.tsx` |
| 实现方式 | 不引入物理引擎；使用纯函数模拟抛竿、咬钩、提竿、张力、断线和跑鱼 |

## 目标

- 让垂钓从「点击按钮拿奖励」升级为可观察、可操作、可失败、可成长的完整玩法。
- 玩家能看见主角本人完成瞄准、抛竿、等待、提竿、搏鱼和收获处理。
- 鱼种结果由钓点、时间、天气、水层、装备和饵料共同决定。
- 失败必须有可理解原因，例如惊鱼、提竿太晚、低张力跑鱼或高张力断线。
- 图鉴记录鱼种、最大重量、捕获次数、放生次数和首次捕获时间。
- 保留桌面键鼠、按钮点击和移动触控路径。
- 不破坏现有自由探索、NPC、天灯、风铃、海螺、开车和移动端按钮。

## 非目标

- 第一版不新增后端接口、交易系统、商城、排行或多人系统。
- 第一版不做任务链、钓鱼等级、稀有成就墙或复杂经济循环。
- 第一版不要求一次性制作完整商业级钓鱼动画资产；若主角 GLB 缺少专用钓鱼动画，可以用程序化上半身动作和挂点过渡，但画面上必须是主角本人。
- 第一版不引入物理引擎、复杂流体模拟或真实鱼群 AI。
- 不把垂钓改成独立页面；它仍是岛屿自由探索中的系统玩法。

## 系统架构

完整垂钓系统拆成规则层、角色动作层、场景反馈层、HUD 层和存储层。

| 模块 | 建议文件 | 职责 |
|---|---|---|
| 状态机与公共类型 | `frontend/src/lib/fishingSystem.ts` | 定义 `FishingPhase`、一次垂钓会话、失败原因、输入事件和结果 |
| 鱼种生态 | `frontend/src/lib/fishingSpecies.ts` | 定义鱼种、稀有度、重量范围、出现条件、挣扎强度和图鉴文案 |
| 装备饵料 | `frontend/src/lib/fishingGear.ts` | 定义鱼竿、鱼线、饵料、加成、耐受范围和默认装备 |
| 模拟规则 | `frontend/src/lib/fishingSimulation.ts` | 纯函数处理抛竿距离、鱼池筛选、咬钩概率、提竿判定、张力变化、断线/跑鱼 |
| 本地存储 | `frontend/src/lib/fishingStorage.ts` | 版本化读写图鉴、统计、当前装备和饵料余量 |
| 主角动作路由 | `frontend/src/lib/protagonistAction.ts` 与 `ExploreMode.tsx` | 扩展钓鱼动作状态，并让 `hero` 优先播放或程序化表现钓鱼动作 |
| 场景反馈 | `ExploreMode.tsx` 中拆出的局部组件，后续可迁移 | 渲染鱼竿、鱼线、浮漂、落点、水花、鱼影和搏鱼反馈 |
| HUD | 新增 `FishingHud` 局部组件或独立组件 | 装备选择、蓄力、提竿提示、张力条、结果卡和图鉴入口 |

`ExploreMode.tsx` 只负责把玩家位置、环境、输入、主角引用和视觉组件接起来；垂钓规则不得散落在大组件内部。

## 玩法状态机

```text
idle -> gear -> aim -> cast -> waiting -> hook -> fight -> result -> idle
                         |        |        |       |
                         |        |        |       -> line_broken
                         |        |        -> fish_escaped
                         |        -> no_bite / cancel
                         -> bad_cast
```

| 阶段 | 玩家看到什么 | 玩家操作 | 结果 |
|---|---|---|---|
| `idle` | 靠近海湾钓点，出现钓具入口 | 打开垂钓 | 进入装备选择或沿用上次装备 |
| `gear` | 鱼竿、鱼线、饵料简面板 | 选择装备和饵料 | 进入瞄准 |
| `aim` | 主角持竿，水面显示落点预览 | 左右调整落点 | 进入蓄力抛竿 |
| `cast` | 主角蓄力，鱼线弧线和落点变化 | 按住蓄力，松开抛出 | 落点太差则 `bad_cast`，否则进入等待 |
| `waiting` | 浮漂漂动，水面有轻微鱼影和水纹 | 观察浮漂，可取消 | 根据鱼池和饵料决定是否咬钩 |
| `hook` | 浮漂轻点、晃动、猛沉 | 猛沉短窗口内提竿 | 太早惊鱼，太晚跑鱼，命中进入搏鱼 |
| `fight` | 主角拉竿，鱼线绷紧，张力条波动 | 收线、放线、稳竿 | 鱼体力归零上鱼；红区太久断线；低张力太久跑鱼 |
| `result` | 主角展示收获或放生，图鉴更新 | 收藏、放生、再钓一次 | 回到 `idle` |

## 主角动作要求

垂钓人物必须是当前可玩主角，不允许用临时小人、NPC 或只靠 HUD 表现。

### 默认主角

- 默认验收对象是 `hero`，即 `xyshz_rigged.glb?v=6`。
- 鱼竿绑定主角右手或最接近的手部骨骼/节点。
- 鱼线从鱼竿竿尖连接到浮漂或鱼的位置。
- 主角朝向必须跟随落点和鱼线方向。

### 动作状态

建议扩展角色动作类型：

```ts
type FishingActionClip =
  | "FishingAim"
  | "FishingCast"
  | "FishingHook"
  | "FishingFight"
  | "FishingReel"
  | "FishingResult";
```

如果 `xyshz_rigged.glb` 暂无这些动画片段，第一阶段使用程序化表现：

- `aim`：主角停步，身体转向水面，右臂持竿。
- `cast`：上半身后仰再前甩，鱼线弧线飞向落点。
- `hook`：短促上提动作，浮漂下沉同步水花。
- `fight`：主角双手拉竿，身体轻微后倾，鱼线张力越高竿身弯曲越明显。
- `result`：展示鱼或放生，保留治愈反馈。

动作优先级应高于普通移动、挥手、吹笛和坐下。进入垂钓状态后锁定移动，取消或完成后恢复普通动作路由。

### 备选角色

低性能设备或用户切换到 `guardian`、`pocoyo`、`avatar` 时可以有降级表现，但不能影响 `hero` 的验收标准。若备选角色暂不支持完整挂点，至少应隐藏备选不足并提示切回主角，避免显示错误人物钓鱼。

## 鱼种生态

第一版放 5 种鱼，足以验证生态、稀有度和搏鱼手感。

| 鱼种 | 稀有度 | 偏好条件 | 搏鱼手感 | 图鉴定位 |
|---|---|---|---|---|
| 银湾小鱼 | 常见 | 浅滩、白天、普通饵 | 张力平稳，教学友好 | 第一次成功钓鱼的安全结果 |
| 潮声鲷 | 常见 | 中距、黄昏、贝肉 | 短促挣扎，容易跑鱼 | 体现提竿和控张力 |
| 雨鳞鱼 | 少见 | 雨天、深湾 | 张力波浪更大 | 让天气影响可被感知 |
| 雾灯鳗 | 稀有 | 雾天或夜晚、礁石 | 会突然反向冲刺 | 体现鱼类行为差异 |
| 星海鱼 | 传说 | 夜晚、星屑饵、远投 | 张力窗口窄，成功反馈强 | 与心屿奇幻氛围连接 |

### 生态输入

| 输入 | 来源 | 影响 |
|---|---|---|
| 钓点 | 玩家所在海湾区域和落点 | 决定浅滩、礁石、深湾、夜潮鱼池 |
| 时间 | 当前探索环境视觉或游戏内昼夜 | 改变夜间、黄昏、白天鱼权重 |
| 天气 | `ExploreEnvironment` | 改变雨天、雾天、风天鱼权重和张力波动 |
| 水层/距离 | 蓄力抛竿结果 | 近岸、中距、远投影响鱼池 |
| 饵料 | 当前装备 | 增加目标鱼权重，降低无咬钩概率 |

## 装备与饵料

第一版采用 3 类鱼竿、3 类鱼线、3 类饵料。

| 类型 | 条目 | 作用 |
|---|---|---|
| 鱼竿 | 竹影竿、海风竿、星潮竿 | 影响抛投稳定、最大鱼重、张力绿区宽度 |
| 鱼线 | 细线、韧线、星丝线 | 影响断线阈值和张力恢复速度 |
| 饵料 | 普通饵、贝肉、星屑饵 | 改变鱼种权重和稀有鱼出现条件 |

第一版可以不做消耗型背包经济，但饵料余量仍可本地保存，用于后续扩展。若不想一开始做库存压力，可把普通饵无限，贝肉和星屑饵通过探索互动少量获得。

## 操作设计

### 桌面端

| 阶段 | 操作 |
|---|---|
| 瞄准 | `A/D` 或鼠标移动微调方向 |
| 蓄力抛竿 | 按住 `Space` 或鼠标主键蓄力，松开抛出 |
| 提竿 | 浮漂猛沉时按 `Space` 或点击提竿 |
| 搏鱼 | 按住收线提高张力，松开放线降低张力；方向键/鼠标小幅稳竿 |
| 结果 | 点击收藏、放生、再钓一次 |

### 移动端

| 阶段 | 操作 |
|---|---|
| 瞄准 | 底部按钮区域左右拖动 |
| 蓄力抛竿 | 长按蓄力，松手抛出 |
| 提竿 | 浮漂猛沉时轻触提竿按钮 |
| 搏鱼 | 按住收线，松开放线；左右滑动稳竿 |
| 结果 | 底部大按钮收藏、放生、再钓一次 |

### 视觉与声音反馈

- 抛竿：鱼线弧线、落点预览、入水水花。
- 等待：浮漂轻晃、水纹、鱼影试探，避免只靠倒计时。
- 咬钩：浮漂先轻点，再明显下沉；此时进入提竿窗口。
- 搏鱼：鱼线绷紧、竿身弯曲、水面冲刺、张力条文字提示。
- 成功：主角收线展示鱼，图鉴闪光，精灵可触发发现反应。
- 失败：明确原因，如「提早了，鱼被惊走」「线绷太久断了」「线松了，它挣脱了」。

## 数据与持久化

第一版使用版本化本地存储：

```ts
interface FishingSaveV1 {
  version: 1;
  selectedRodId: string;
  selectedLineId: string;
  selectedBaitId: string;
  baitInventory: Record<string, number>;
  codex: Record<string, FishingCodexEntry>;
  stats: {
    totalCatches: number;
    totalReleased: number;
    totalEscaped: number;
    totalLineBreaks: number;
  };
}

interface FishingCodexEntry {
  speciesId: string;
  firstCaughtAt: number;
  catchCount: number;
  releaseCount: number;
  maxWeight: number;
}
```

存储键建议为 `xy_fishing_v1`。旧的 `xy_catch` 可以保留兼容，用于展示历史总数或迁移到 `stats.totalCatches`。

## 错误处理与边界

| 场景 | 处理 |
|---|---|
| 玩家离开钓点 | 取消当前会话，隐藏鱼竿/浮漂/鱼线，恢复主角移动 |
| NPC 或其它互动靠近 | 垂钓状态优先锁定，取消后才显示其它互动 |
| 抛竿落点无效 | 显示水花或落空提示，不进入咬钩 |
| 没有鱼咬钩 | 超时后提示换落点、换饵或等天气变化 |
| 过早提竿 | 进入惊鱼失败 |
| 过晚提竿 | 进入跑鱼失败 |
| 张力过高 | 红区持续超过阈值断线 |
| 张力过低 | 鱼距离拉大，持续过久跑鱼 |
| localStorage 不可用 | 玩法继续，图鉴和统计不保存，并静默降级 |
| 低性能设备 | 减少鱼影、水花和线条细节，但主角本人垂钓不降级 |

## 测试与验证

实现时采用 TDD（测试驱动开发，先写失败测试，再写最小实现让测试通过）。

| 验证项 | 标准 |
|---|---|
| 规则单元测试 | 鱼池筛选、权重、抛竿距离、提竿窗口、张力变化、断线/跑鱼边界可重复测试 |
| 存储测试 | `xy_fishing_v1` 可读写、可容错、可从空存储初始化 |
| 主角动作测试 | `hero` 在垂钓阶段进入钓鱼动作状态，鱼竿挂到主角而不是临时人物 |
| 结构测试 | 规则位于 `frontend/src/lib`，`ExploreMode.tsx` 不承载大段模拟逻辑 |
| 回归测试 | 现有 `fishingRhythm.test.mjs`、`fishingBobberModel.test.mjs` 和角色动作相关测试更新后通过 |
| 构建 | `cd frontend && npm run build` 通过 |
| 全量前端测试 | `cd frontend && npm test` 通过 |
| 视觉手测 | 桌面和移动视口下验证主角抛竿、浮漂下沉、提竿、搏鱼、断线/跑鱼、图鉴结果 |

## 验收标准

- 玩家靠近海湾钓点后可以进入完整垂钓系统。
- 画面中执行垂钓的人物是当前可玩主角，默认是 `hero` / `xyshz_rigged.glb?v=6`。
- 主角能完成瞄准、蓄力抛竿、提竿、搏鱼和结果展示。
- 鱼竿与鱼线从主角手部和竿尖出发，不是漂浮在 HUD 或世界坐标里的孤立道具。
- 抛竿距离影响水层，水层影响鱼池。
- 浮漂动作替代现有抽象光圈，玩家通过浮漂和水面判断提竿时机。
- 搏鱼阶段需要控制张力；高张力可断线，低张力可跑鱼。
- 鱼种由钓点、时间、天气、饵料和距离共同决定。
- 图鉴记录至少包含鱼种、最大重量、捕获次数、放生次数和首次捕获时间。
- 失败结果有明确原因，并能重新开始垂钓。
- 不破坏现有自由探索、NPC、天灯、风铃、海螺、开车、移动端摇杆和按钮。

## 实施切片

由于这是高风险变更，建议后续实施分 4 个阶段：

1. 规则核心：新增鱼种、装备、模拟、存储纯函数和红绿测试。
2. 主角动作接入：扩展动作路由，锁定移动，鱼竿绑定主角手部，鱼线跟随竿尖和浮漂。
3. 场景反馈：落点、水花、浮漂动作、鱼影、张力表现和失败反馈。
4. HUD 与图鉴：装备饵料面板、张力条、结果卡、图鉴和移动端适配。

每个阶段都要保持现有测试通过，且每次只暂存本阶段相关文件。

## 风险与处理

| 风险 | 影响 | 处理 |
|---|---|---|
| `ExploreMode.tsx` 继续膨胀 | 后续维护困难 | 规则和存储先拆 `src/lib`，组件只接线 |
| 主角缺专用钓鱼动画 | 真实感不足 | 第一阶段用程序化上半身和挂点过渡，后续再补 GLB 动画 |
| 鱼竿挂点难以稳定定位 | 鱼竿漂浮或穿模 | 优先找手部骨骼/节点；找不到则建立主角局部空间挂点并测试 |
| 张力玩法过难 | 破坏治愈节奏 | 默认绿区宽、鱼体力低，稀有鱼再提高难度 |
| 生态数值失控 | 鱼种结果不可解释 | 第一版只做 5 种鱼、3 类钓点、3 级装备、3 类饵料 |
| 移动端误触 | 操作挫败 | 底部大按钮、长按/松手语义统一，并保留取消 |
| 本地存储迁移出错 | 图鉴丢失 | 使用 `version` 字段，读失败回默认值，不覆盖未知数据 |

## 术语说明

| 术语 | 说明 |
|---|---|
| State Machine | 状态机，把玩法拆成固定阶段，避免多个布尔状态互相冲突 |
| HUD | Head-Up Display，游戏界面上的操作提示、按钮和状态条 |
| GLB | 一种 3D 模型文件格式，当前主角、浮漂和岛屿物件都使用这类资源 |
| 挂点 | 把道具绑定到模型某个骨骼或节点上的位置，例如把鱼竿挂到主角右手 |
| 张力 | 鱼线承受的拉力；过高会断线，过低鱼会挣脱 |
| 鱼池 | 某个环境条件下可能出现的鱼种集合 |
| 权重 | 随机选择时的概率倾向，权重越高越容易出现 |
| localStorage | 浏览器本地存储，用于第一版保存图鉴、装备和统计 |
| TDD | 测试驱动开发，先写失败测试，再写实现代码，最后保持测试通过后重构 |
