# 《心屿》需求文档

> **项目定位**：AI 驱动的情绪世界生成游戏（情感陪伴 / 治愈叙事）
> **赛事背景**：腾讯云黑客松「AI CAN DO IT 极限游戏开发挑战赛」· 社会价值「小红花」赛题（含适老化 / 独居人群方向）
> **文档版本**：v5.0（完整自包含版，依据仓库实际实现回写）
> **编写日期**：2026-06-20
> **一句话边界声明**：心屿提供情感陪伴体验，不是心理咨询或医疗服务，不做诊断，也不承诺疗效。

## 1. 项目概述

### 1.1 项目背景

赛事以 **AI 驱动的游戏创作**为核心。《心屿》面向大学生与年轻职场人常见的焦虑、疲惫、孤独与压力，并刻意延伸到**适老化 / 独居人群**：把 AI 文本理解、情绪记忆、确定性环境状态机与叙事生成结合起来，构建一座低门槛、无羞耻感、可持续回访的情绪岛屿。

> **AI 把用户情绪转化为可持续生长的游戏世界。每一次倾诉都会改变岛屿，每一次返回都能看见自己曾经走过的心理轨迹；说不出话、写不动字，也能来岛上坐一会儿。**

### 1.2 项目愿景

打造一座会倾听、会记忆、会生长、也会主动回望的 AI 心灵岛屿，让用户在压力和孤独时获得一个低门槛、可删除、可持续回访的情绪出口。

### 1.3 Slogan 与一句话

> **心屿不教你变得更好，它只是记得你来过、陪你长出形状。**

它不是工具、不是教练、不是咨询师，而是一个允许你「什么都做不好、什么都说不出」时也能来坐一会儿的地方。

### 1.4 项目类型

| 维度 | 内容 |
|---|---|
| 赛事方向 | AI 驱动游戏创作 · 社会价值赛题 |
| 游戏类型 | 治愈叙事游戏、情绪陪伴游戏、轻量交互式体验 |
| 社会价值 | 青年与独居人群心理压力缓解、低门槛 / 非语言情绪表达、代码级安全护栏 |
| 目标平台 | Web 端优先，兼容移动端浏览器；支持 PWA 安装与离线；3D 体验随设备能力自动分级 |
| 核心玩法 | 倾诉 / 静默 / 写一个字、观看岛屿变化、留下并刻写物件、与岛屿对话、自由探索与开车环岛、回看情绪轨迹 |
| 胜利条件 | 不是变强，是**被记得** |

## 2. 用户与场景

### 2.1 目标用户

| 用户类型 | 特征 | 核心诉求 |
|---|---|---|
| 高校学生（尤其低年级） | 学业、竞赛、就业、社交压力高，刚进入新环境 | 想找一个低压力的情绪出口 |
| 年轻职场人 | 加班、转型、孤独感、成就焦虑 | 希望被理解，也希望看见自己慢慢变好 |
| 独居 / 适老化人群 | 日常缺乏被惦记的感受，交互能力或意愿有限 | 极低门槛的陪伴：一句话、一个字，甚至只是坐着 |
| 黑客松评委 | 关注 AI 中心性、游戏性、技术完整度、社会价值 | 希望看到 AI 真正驱动游戏世界且边界清晰 |

### 2.2 典型使用场景

| 场景 | 用户行为 | 系统回应 |
|---|---|---|
| 说不出话 | 进岛后不打字，点「静默坐岛」 | 岛屿陪坐 N 秒，留下一枚静默贝壳作为「我来过」的证据，不写记忆库 |
| 写不动整句 | 描红写一个心境字「累」 | 岛屿结合字义 + 书写动力学（笔速 / 停顿 / 抖动）读出情绪，刻成一块心境石 |
| 焦虑倾诉 | 输入「明天答辩，我很紧张」 | 识别焦虑，岛屿起雾，高强度时先引导 4-7-8 呼吸，再给叙事 |
| 疲惫回访 | 多次记录「很累」 | 岛屿夜色加深，灯塔逐渐亮起；岛屿主动低语提示「你上次也带着疲惫来过」 |
| 开心记录 | 输入「今天拿奖了，很开心」 | 岛屿长出花、音乐变明亮，可选择留下一枚物件并刻字 |
| 想多聊几句 | 在主页「说给岛屿」里多轮对话 | 岛屿用治愈语气回应，必要时自主翻看你过去的记忆再回答 |
| 自由探索 | 进入 3D 岛屿漫游 / 上车环岛 | 拾取心灵印记、种花、放天灯、垂钓、奏风铃，触发岛屿奇遇彩蛋 |
| 久未回岛 | 离开 48 小时以上再进入 | 岛屿递上一封「我替你看着」的离岛信件 |
| AI 自我纠偏 | 隔天回岛 | 岛屿主动承认「我昨天那句说得不太准确」，给出修正信 |
| 高风险表达 | 输入强烈绝望 / 自伤相关内容 | 暂停普通叙事、不给选择卡，显示安全提示与求助引导 |
| 答辩演示 | 一键注入演示轨迹 | 时光机逐步「荒岛生长」，年报 / 修正信 / 离岛信件依次展示 |

## 3. 产品定位与核心创新

### 3.1 定位

> **AI 驱动的情绪世界生成游戏。**

不只生成一段安慰文本，而是把用户情绪转化为三层游戏结果：

| 层级 | 说明 | 用户感知 |
|---|---|---|
| 即时回应层 | 识别情绪并生成治愈叙事 + 心灵印记 | 「它听懂了我现在的心情」 |
| 记忆连续层 | 检索历史情绪并主动回访（低语 / 离岛信 / 修正信 / 年报 / 对话） | 「它记得我之前发生过什么」 |
| 世界生成层 | 根据情绪历史改变岛屿状态、物件与成长等级 | 「我的心情真的改变了这座岛」 |

### 3.2 核心创新点

| 创新点 | 说明 | 比赛价值 |
|---|---|---|
| 心象岛屿生长系统 | 情绪记忆转化为岛屿元素（雾 / 灯塔 / 花 / 星光 / 雨痕……）与 1-5 级成长，叠加玩家留下的物件 | 游戏性与视觉表现 |
| 五感信使·可解释编排 | 情绪 / 记忆 / 环境 / 叙事 / 安全五张卡片「逐个抵达」，让人看见 AI 如何一步步读懂你 | 可解释性，而非黑箱 |
| 非语言情绪入口 | 静默坐岛、写一个字、呼吸仪式——承认「说不出也算说话」 | 独占别人不服务的人群，强社会价值 |
| 岛屿主动回望 | 主动低语、离岛信件、修正信、年报、时光机回望 | 持续陪伴与回访动力 |
| 治愈知识库驱动的 AI 对话 | 与岛屿多轮对话 / 常驻助手 / 陪伴精灵共用单一「治愈知识库」语气与边界，且可自主调用记忆工具 | AI 中心性与一致的治愈口吻 |
| 沉浸式 3D 与自由探索 | 真 3D 岛屿、可漫游、可开车环岛，配分层情绪声景 | 游戏完整度与体验 |
| 双端安全网 | 输入端双触发（强度阈值 + 关键词黑名单）、输出端复检防复述 | 代码级社会责任硬证据 |
| 隐私可删除 | 应用内一键删除身份全部后端痕迹（记忆 / 物件 / 私房话 / 向量） | 面向脆弱人群的可验证承诺 |

### 3.3 AI 边界的诚实化口径（重要）

心屿对「AI 用在哪、怎么用」有一条明确原则：**该用 AI 用 AI、该用规则用规则**。具体分两条链路，对外都不夸大：

**① 情绪反思主链路（确定性优先）。** 一次倾诉走的是一条**线性可解释管道** `_reflect_pipeline`（`backend/app/main.py`）。

- 真正调用 LLM 的只有**两步**：情绪分析（`emotion_service`）与叙事表达（`narrative_service`）。
- 其余阶段都是**确定性函数**：安全边界纯规则（`safety_service`）、场景与岛屿成长纯 Python 状态机（`scene_map` / `island_state_service`）、记忆检索为 pgvector + PostgreSQL。
- `agent_trace` 的五张卡片对用户和评委的价值是**可解释性**，不是宣称五个会自主通信的智能体。对外统一称：**可解释的情感 AI 编排管道**或更口语的**「岛屿用五感聆听你」**。面对情绪脆弱用户，可解释与确定性比「看起来很 AI」更重要。

**② 对话陪伴链路（真·function-calling agent）。** 主页「说给岛屿」多轮对话、常驻 AI 助手、自由探索中的陪伴精灵，使用 `agent_service.py`（DeepSeek function-calling）：模型**自主决定**是否调用工具（`recall_memories` 翻看历史记忆 / `read_island` 读岛屿状态）再作答。这是一个真实的工具调用智能体，但它被**刻意限定在对话场景**，不介入安全攸关的反思主链路。

> 一句话：**情绪反思追求确定与可核验，对话陪伴才放开为自主 agent。** 两条链路都接入同一套「治愈知识库」（见 F28）保证语气与边界一致。

## 4. 术语说明

| 术语 | 解释 |
|---|---|
| 可解释编排管道 | 一次输入按固定顺序流经情绪 / 安全 / 记忆 / 环境 / 叙事各阶段，过程对用户可见 |
| 情绪记忆 | 系统保存的历史心情记录：文本、情绪标签、强度、摘要、叙事、印记和时间 |
| 向量记忆 | 把文本转成向量用于检索语义相似的历史记忆（pgvector，不可用时回退 PostgreSQL 最近记忆） |
| 心象岛屿 / 心象地图 | 情绪在游戏世界中的可视化与历史轨迹 |
| 心灵印记 | 每次普通交互生成的一句 20-60 字短诗 / 语录 |
| 物件（Artifact） | 玩家通过选择卡 / 静默 / 写字在岛上留下的收藏物，可刻字 |
| 私房安慰话 | 用户教给岛屿的、来自重要他人的安慰话，按情绪在叙事后被引用 |
| 治愈知识库（healing_kb） | 单一可信源：岛屿语气 / 倾听原则 / 硬边界 / 各情绪侧重，注入全部 AI 角色的系统提示 |
| 岛屿对话 / 常驻助手 / 陪伴精灵 | 三个 AI 对话入口，由可自主调用记忆工具的 agent 驱动 |
| 岛屿奇遇 | 自由探索中走近触发的彩蛋（小动物 / 夜晚魔法 / 海与水 / 互动仪式），按发现数聚合显示 |
| 无痕模式 | 勾选后本次不写记忆、不留物件、不推进成长——陪你但不留痕 |
| 章节导语（chapter） | 岛屿状态的「去向 / 在等待什么」一句话，描述性、不预测、不承诺好转 |
| 五感信使 | 五张 Agent 卡片的对外说法，强调可解释而非自治 |
| 安全提示 | 检测到高风险表达时停止普通叙事并引导求助 |

## 6. 详细功能需求

### 6.1 F1 情绪入口

#### 描述
用户进岛后通过文字或语音输入当下心情，可短句、长段或口语化。组件：`MoodInput.tsx` + `VoiceInputButton.tsx`。

#### 输入方式
| 方式 | 优先级 | 说明 |
|---|---|---|
| 文字输入 | P0 | 核心，多行文本 |
| 快捷情绪短句 | P0 | 提供示例输入，便于演示 |
| 浏览器语音输入 | P1 | `SpeechRecognition` / `webkitSpeechRecognition`，不支持时降级为不可用 |
| 无痕模式开关 | P1 | 勾选后本次不写记忆、不留物件、不推进成长，前端显示「岛屿不会记得这次」 |

#### 输入限制
| 项目 | 要求 |
|---|---|
| 最大长度 | 2000 字（后端 400 拒绝超长） |
| 空文本 | 禁止提交（后端 400） |
| 用户身份 | 本地匿名 `user_id`（默认 `demo-user`），不注册登录 |
| PII 提醒 | 不鼓励填写真实姓名、手机号、学号、邮箱 |

#### 验收标准
- 可输入文本并提交，空文本不能提交，后端返回明确错误。
- 勾选无痕后，响应 `ephemeral=true`，不产生记忆与物件。

### 6.2 F2 五感信使·可解释编排（导演台）

#### 描述
展示一次输入如何被转化为游戏体验。前端组件 `AgentDirectorPanel.tsx` 以五张卡片逐个点亮，是答辩里的技术可解释窗口。

#### Agent 卡片
| agent | 中文名 | 职责 | 是否调用 LLM |
|---|---|---|---|
| emotion | 情绪分析 Agent | 判断情绪类别、强度、摘要 | ✅ 是 |
| memory | 记忆检索 Agent | 检索最近 / 相似历史记忆 | ❌ pgvector + PostgreSQL |
| environment | 环境推理 Agent | 决定场景与岛屿状态 | ❌ 纯 Python 状态机 |
| narrative | 叙事表达 Agent | 生成治愈叙事与心灵印记 | ✅ 是 |
| safety | 安全边界 Agent | 判断是否触发高风险提示 | ❌ 纯规则 |

#### 输出形态
每张卡片含：agent 名称、中文 label、状态（waiting / running / done）、可读输出摘要（如「识别为疲惫，强度 0.72」「岛屿成长至第 3 级」）。

#### 流式行为
WebSocket `/ws/reflect` 按阶段实时推送，让情绪 / 记忆 / 环境先点亮，再等最慢的叙事——「信使逐个抵达」，而非算完一次性闪过。

#### 验收标准
- 用户提交后导演台按阶段展示五个 Agent。
- 每个 Agent 至少有一条可读摘要。
- HTTP 回退时也能展示最终 `agent_trace`。
- 高风险场景下 safety 卡片显示已触发安全边界。

### 6.3 F3 心象岛屿生长系统

#### 描述
`island_state_service.py`（纯 Python，零 LLM，断网可跑）根据当前输入与历史记忆生成 `island_state`。

#### 字段
```json
{
  "dominant_emotion": "tired",
  "trend": "recovering",
  "growth_level": 3,
  "features": ["river_lamp", "stars", "hammock", "lighthouse"],
  "weather_memory": "clearing_sky",
  "summary": "你的心象岛屿成长到第 3 级，最近以疲惫为主。最近的云层正在慢慢散开，灯塔的光比之前亮了一些。",
  "chapter": "岛屿正从雾季里慢慢走出来——它不催你，只陪你一程一程地走。"
}
```

#### 成长等级规则（按记忆总数）
| 等级 | 条件 | 可见元素数 |
|---|---|---|
| 1 | ≤ 1 条 | 1 个 |
| 2 | 2-4 条 | 1 个 |
| 3 | 5-9 条 | 2 个（灯塔从此常驻） |
| 4 | 10-19 条 | 3 个 |
| 5 | ≥ 20 条 | 3 个 |

#### 趋势判断（基于最近序列，含本次）
| trend | 逻辑 | 文案方向 |
|---|---|---|
| `stormy` | 最近 3 条里 ≥ 2 条高强度（≥0.8）负面 | 风浪较大，多停靠休息 |
| `brightening` | 最近 3 条里 ≥ 2 条正面 | 岛屿正在变亮 |
| `recovering` | 负面但最新强度 < 较早均值 - 0.05 | 云层正在散开 |
| `mixed` | 历史出现 ≥ 4 种不同情绪 | 天气交错 |
| `stable` | 其余（默认） | 平稳潮汐 |

#### 情绪 → 岛屿元素（EMOTION_FEATURES）
| 情绪 | features |
|---|---|
| sad 难过 | rain, lighthouse, tide |
| anxious 焦虑 | fog, rocks, wind |
| tired 疲惫 | stars, hammock, fireflies |
| lonely 孤独 | single_tree, moonlight, distant_boat |
| calm 平静 | sailboat, calm_water, sunrise |
| happy 愉悦 | flowers, butterflies, sunshine |
| angry 愤怒 | cliffs, storm, spray |
| helpless 无助 | faint_light, still_water, shelter |

玩家留下的**物件 key** 会作为持久元素优先合并进 `features`（去重，最多 8 个）。`weather_memory` 由主导情绪与趋势映射（clearing_sky / recent_rain / misty / clear_sky / after_storm / drifting_tide）。

#### 章节导语（chapter）
为成长等级提供「方向感」而非纯计数。**伦理红线：描述性、不评判、不预测、绝不承诺方向必然向好。**

#### 验收标准
- 每次普通提交后返回 `island_state`（含 `chapter`）。
- 记忆数量变化时 `growth_level` 合理变化。
- 不同主导情绪返回不同 `features`，物件能持久叠加。
- 无痕模式下 dominant/trend/weather 仍反映本次，但 `growth_level` 不推进。

### 6.4 F4 AI 治愈叙事 + 心灵印记

#### 描述
`narrative_service.py` → `llm_provider` 根据情绪、强度、意象与历史生成 50-120 字中文叙事 + 20-60 字心灵印记 + 记忆提示。

#### 叙事原则
温柔克制、画面化（岛屿 / 海浪 / 光 / 雨 / 风意象）、有边界（不说「你一定会好」「我能治愈你」）、有连续感（自然体现「岛屿记得你」）、安全优先（高风险时不输出普通故事）。

#### 输出示例
```json
{
  "narrative": "夜深了，岛上的风很轻。你躺在吊床里，星星替你守着今晚……",
  "imprint": "今晚先把自己交给星光，明天的海会替你重新托起帆。",
  "memory_hint": "岛屿记得你上次也带着疲惫来过，但你依然走到了今天。"
}
```

#### 降级与安全网
- 真实模型失败时每个方法自动降级到本地 Mock 模板（关键词驱动，断网可跑）。
- **输出端安全网**：万一 LLM 生成内容命中风险关键词，`_scrub_generated` 降级为最克制的陪伴句。

#### 验收标准
- 普通输入生成叙事、印记、记忆提示；高风险输入叙事为空、印记为 `null`。
- 叙事长度以 50-120 字为主，不含医疗诊断或治疗承诺。

### 6.5 F5 情绪记忆系统

#### 描述
`memory_service.py`（PostgreSQL 关系表）保存每次记录，`vector_memory_service.py` 经 pgvector 语义索引检索相似历史；底层连接与建表在 `db.py`。

#### 存储字段（memories 表）
`id`、`user_id`、`text`、`emotion`、`intensity`、`summary`、`narrative`、`imprint`、`created_at`。

#### 检索机制
| 方式 | 说明 |
|---|---|
| 最近记忆 | 取最近 N 条用于趋势 / 地图 / 岛屿状态 |
| 向量相似记忆 | pgvector 检索语义相似情绪（默认取 3 条），失败自动回退 PostgreSQL 最近 |
| 计数 | `count_by_user` 决定成长等级 |

#### 隐私边界
- 不注册；不保存手机号 / 学号 / 邮箱 / 真实姓名。
- `user_id` 仅用于本地 Demo 隔离。
- 提供按 `user_id` 彻底删除后端记忆 / 物件 / 私房话 / 向量的能力（见 F23）。

#### 验收标准
- 不同 `user_id` 记忆互相隔离。
- 删除后历史列表与地图清空，pgvector 向量经 ON DELETE CASCADE 随记忆一并清除，删除关系记忆永不被阻断。

### 6.6 F6 心象地图 / 记忆之树

### 6.7 F7 多模态环境反馈

#### 描述
根据情绪与岛屿状态切换场景、音乐与动效。

#### 场景（scene_map.py）
8 种情绪 × 3 个强度档 = **24 个场景**。强度分档：< 0.58 低 / < 0.78 中 / ≥ 0.78 高。`scene` 字段含 `time`（dawn/day/dusk/night）、`weather`（clear/light_rain/rain/fog/storm）、`palette`（如 `tired_mid`）、`music`、`imagery`。

#### 视觉表现
- **真 3D 岛屿**（`Island3D.tsx`，react-three-fiber，详见 F24）：地形、植被、生物、灯光、后处理随情绪平滑变色。
- **2D 底图 / 降级**（`IslandScene.tsx`）：24 张本地预设场景图（8 情绪 × 3 强度档）作底图，叠加 CSS 渐变天空 / 海面、SVG 岛屿与天气粒子，无需 WebGL；3D 不可用时即以此为准。
- 元素由 `island_state.features` 驱动，按 `islandMeta.ts` 的 FEATURE_META 渲染图标与位置。
- 叙事完成时有「生长瞬间」光效。

#### 音频（musicMap.ts / ambience.ts / locationAmbience.ts / sfx.ts）
后端 `scene.music` 现已直接返回**情绪键**（sad/anxious/tired/lonely/calm/happy/angry/helpless/default），前端映射到 `/public/audio/{emotion}.m4a`（Kevin MacLeod，CC-BY 4.0，附 CREDITS.md）。旧 palette / 曲名键保留兼容别名。BGM 之上再叠两层环境声景：随情绪切换的**氛围底噪**（`ambience.ts`：海浪 / 雨 / 夜虫 / 篝火 / 林风 / 溪流 / 晨鸟）与自由探索时随区域切换的**位置环境音**（`locationAmbience.ts`：洋 / 湾 / 溪 / 篝火 / 山 / 林 / 村）。交互音效 `sfx.ts` 采取**采样优先**：命中 `samples.ts` 预载的真实采样即播，未命中或断网回退 Web Audio 合成。新增环境 / 采样素材取自 Wikimedia Commons（CC0 / PD / CC-BY(-SA)），署名见 CREDITS.md 与 meta.json。

### 6.8 F8 安全边界（双触发 + 输出端复检）

#### 描述
`safety_service.py` 纯规则、无 LLM。这是公益赛道里可逐行核验的硬证据。

#### 输入端双触发（任一即触发）
1. **强度阈值**：情绪 ∈ `{sad, angry, helpless, anxious}` 且强度 ≥ **0.85**。
2. **关键词黑名单**：命中 `SAFETY_KEYWORDS`（中英双语 + 委婉 / 网络黑话），**独立于强度**——专修「平静措辞的自伤表达因强度低而漏过」漏洞。`has_risk_keyword` 还做全角转半角 + 去标点的「紧凑形」匹配，抵御「想 死」「自。杀」等混淆。

#### 触发即让位
普通叙事整段暂停（`narrative=""`、`imprint=null`、`choices=[]`），改为统一求助引导文案，引导联系可信任的人或拨打**全国统一心理援助热线 12356** / 当地紧急电话。

#### 输出端复检防复述
`memory_is_high_risk` 在所有「回看历史」的功能（低语 / 年报 / 修正信 / 时光机 / 对话）里对历史原文二次复检，把高风险记忆从素材里剔除，防止 LLM 复述用户过去的自伤原文。

#### 安全提示原则
不诊断、不承诺平台能解决危机、鼓励求助、本地化资源。

#### 验收标准
- 高风险输入不生成普通叙事，返回 `safety.triggered = true`。
- 平静措辞但含风险关键词的输入也能稳定触发。
- 安全提示永不被「呼吸 / 沉浸 / 氛围 / 对话」交互盖住或延后。

### 6.9 F9 岛屿回应选择卡 + 物件收藏

#### 描述
`island_ritual_service.py` + `artifact_service.py`。普通叙事后给 3 张「面对情绪的方式」选择卡（`IslandChoiceCards.tsx`），玩家选择后在岛上留下一枚持久物件。

#### 选择卡结构（IslandChoice）
`id`、`stance`（面对方式，如「让难过被看见」）、`ritual`（岛屿仪式，如「在岸边点一盏河灯」）、`artifact`（物件 key）、`reply`（选择后的反馈叙事）、`rare`（是否此刻限定）。

#### 此刻限定稀缺卡
当 trend ∈ {recovering, brightening} 时插入「破晓的光」稀缺卡（`rare=true`），替换一张普通卡，保持 3 张。

#### 物件
约 16 类（lantern / paper_boat / night_flower / shell / star_wish / river_lamp / stone_cairn / kite / feather / candle / sail / leaf_note / bonfire / bloom / silent_shell / glyph_stone），各有 emoji 图标与中文 label，存于独立 `artifacts` 表。`GET /api/artifacts` 查询，`POST /api/island/act` 留下。

#### 边界
高风险或无痕模式下不给选择卡（物件是持久的，与「不留痕」承诺冲突）。

### 6.10 F10 给未来自己刻字 + 时光回信

#### 描述
玩家可在一枚已留下的物件上刻一句给未来自己的话（≤ 80 字，`POST /api/artifacts/{id}/inscribe`）。

#### 时光回信
当用户再次进岛时，岛屿主动低语可能把 ta **亲手刻下的那句话**带回来：「你曾在 X 上写下：『……』——这句话岛屿一直替你留着。」系统自动刻字（静默贝壳 / 心境石）不计入手写句，只取用户真正手写的。

### 6.11 F11 私房安慰话回响

#### 描述
`phrase_service.py`。用户把来自重要他人的安慰话教给岛屿（绑定某情绪，`POST /api/phrases`，组件 `IslandPhrases.tsx`）。当再次出现同类情绪时，岛屿在叙事后以引号附议该话（`echo_phrase`）。AI 在此**退后一步**，引用的是用户支持网络里真实的人的话，而非 AI 生成。

#### 约束
content ≤ 120 字、attribution ≤ 24 字、emotion 必须在 8 类白名单内；可增删（`DELETE /api/phrases/{id}` 软删）。高风险与无痕模式下不回响。

### 6.12 F12 静默坐岛

#### 描述
`SilentMode.tsx` + `POST /api/silent/companion`。用户什么都不说，岛屿陪坐 N 秒（默认 30s），留下一枚静默贝壳作为「我来过」的证据。**不写记忆库**（没有倾诉文本），把静默时长温柔地刻在贝壳上。这是「说不出也算说话」的核心入口，承认沉默是合法情绪状态。

### 6.13 F13 写一个字给岛屿（手写一字读心）

#### 描述
`GlyphCanvas.tsx` + `POST /api/glyph`。用户描红写下一个心境字（字确定，无需 OCR），前端捕捉**书写动力学**（笔速 / 时长 / 笔画数 / 停顿数 / 抖动），后端 `provider.read_glyph` 结合字义 + 动力学读出情绪与强度，刻成一块心境石，多块积累成石林。

#### 可书写心境字（GLYPH_CHARS）
累(tired)、沉(sad)、空(lonely)、撑(helpless)、涌(anxious)、稳(calm)、静(calm)、暖(happy)。

### 6.14 F14 呼吸仪式

#### 描述
`BreathingRitual.tsx`。当识别到高强度负面情绪（anxious/sad/angry/helpless 且强度较高）且**未触发安全边界**时，叙事前先引导一轮 4-7-8 呼吸着陆（吸气 / 屏息 / 呼气 × 3）。安全优先：若触发自伤风险，跳过呼吸直达安全提示——求助永不被呼吸阻挡。

### 6.15 F15 岛屿主动低语

### 6.16 F16 离岛信件 / 欢迎回来

#### 描述

### 6.17 F17 岛屿修正信（AI 自我纠偏）

#### 描述
`IslandRevision.tsx` + `GET /api/island/revision`。`revision_service.py` 在 12-72 小时窗口内挑一条昨日叙事，LLM 回看后主动承认「我那句说得不太准确」。`kind` ∈ {too_heavy（说重了）, too_light（说浅了）, off_topic（没说到点）}。这是反思链路里**唯一一处多步 AI 协作（生成 → 规则批判 → 必要时降级）**，且完全为安全 / 诚实服务。`force=1` 演示开关放宽时间窗。

### 6.18 F18 岛屿年报

#### 描述
`IslandLetter.tsx` + `POST /api/island/letter`。LLM 读全部历史记忆 + 物件，写一封约 180-220 字、第二人称、不规劝的温柔短信，含一个观察到的情绪 / 行为模式与真实物件引用（只引用列表里的物件，绝不编造）。无记忆时返回空。展示「长窗上下文」能力。

### 6.21 F21 情感语音合成 TTS

### 6.24 F24 真 3D 岛屿 + 自由探索

#### 描述
- **Island3D.tsx**（懒加载）：react-three-fiber 真 3D 岛屿——默认以整景 GLB `xy_scene_island.glb` 实时 cel-shading 渲染（`toonifyIsland` 按材质名套 `MeshToonMaterial` + 自发光保辉，雪峰入镜、水下辉光垂入水线倒影），并漂浮云朵 / 飞鸟 / 海龟 / 暖阳等背景生灵与天体；灯光与（高性能档）Bloom / God Rays / Vignette 后处理，所有材质颜色随情绪平滑插值；`?island=proc` 可切回程序化地形（实例化草 / 树 / 花 / 礁石 / 鸟群 / 鲸鱼）兜底。
- **ExploreMode.tsx**（懒加载）：可漫游岛屿——卡通着色角色（WASD / 方向键 / 触摸摇杆），GLB 地标（灯塔 / 风车 / 神社 / 鸟居 / 码头 / 船 / 鲸石 / 贩卖机）、村落与海岸地形，可拾取的愿望灯 / 心灵印记 / 漂流瓶 / 河灯（即「拾心灵印记」玩法）。角色与树 / 房屋 / 地标之间有空间网格碰撞，不可穿模。
- **分级与降级**：`useSkin3d.ts` + `perfTier.ts` + `webgl.ts` 按 WebGL 支持与设备性能分高 / 低档，移动端与软件渲染强制低档；无 WebGL 时回退 2D `IslandScene`。首屏对写实重模型做 defer 延后与独立 Suspense，并对 three 做分包，缩短上岛加载时间。
- 模型资源在 `frontend/public/models/`（本版本约 118 个 GLB，由 Blender 离线生成，构建脚本见 `blender/`，原画备份在 `_model_originals/`）。

> 自由探索中的开车、花田 / 天灯 / 垂钓 / 风铃玩法与岛屿奇遇彩蛋分别见 F30、F31。

### 6.26 F26 流式与幂等韧性

#### 描述
- 前端默认走 `/ws/reflect`，流式失败 / 超时回退 `POST /api/reflect`。
- 客户端每次提交带 `request_id`；后端用 LRU 缓存 + per-id 锁做**幂等去重**：WS 超时回退 HTTP 时命中缓存，绝不二次执行 / 重复落库。
- `__final__` 事件在 save 完成后先落缓存，即便随后推送失败也不重复落库。
- WS 握手在 accept 前做 Origin 同源校验（CORSMiddleware 不覆盖 WS）。

### 6.27 F27 PWA 离线体验

#### 描述
应用是可安装的 PWA：`frontend/public/manifest.webmanifest`（名称 / 图标 / 主题色 / 独立窗口）、`sw.js` Service Worker、`offline.html` 离线兜底页，由 `src/lib/registerSW.ts` 在 `main.tsx` 注册。

#### 行为

- Service Worker 升级走标准生命周期，新版本可平滑接管。
- 离线时依赖后端的功能（反思 / 对话 / TTS）按既有降级策略提示不可用，不阻断已缓存内容浏览。

### 6.28 F28 岛屿对话 / 常驻助手（function-calling agent）

#### 描述
在主页提供两个 AI 对话入口，均由 `agent_service.py` 驱动（DeepSeek function-calling），模型**自主决定**是否调用工具再作答：
- **说给岛屿（多轮对话）**：`IslandChat.tsx` + `POST /api/chat`，用 `ToolChatAgent` 带上整段对话历史（≤ 60 轮）多轮回应。
- **常驻 AI 助手**：`IslandAssistant.tsx` + `POST /api/agent/ask`，用户随时提问，`AgentReflectionService` 调工具作答。

#### 可用工具
| 工具 | 作用 |
|---|---|
| `recall_memories` | 检索该用户的历史情绪记忆（语义 / 最近） |
| `read_island` | 读取当前岛屿状态与成长 |
| `list_recent`（对话场景额外提供） | 列出最近记忆，支撑「你上次说过……」类回应 |

返回含 `reply` / `answer`、`tools_used`（本轮实际调用过的工具，可见可解释）、`safety`。

#### 治愈知识库（healing_kb.py）
单一可信源：岛屿语气、倾听原则、硬边界、各情绪侧重，经 `compose_system_prompt` 注入**全部 AI 角色**（叙事 / 低语 / 对话 / 常驻助手 / 陪伴精灵），保证语气与边界一致。`GET /api/health` 暴露 `healing_kb` 版本号；改 KB 需重启后端。

#### 边界
- 对话同样接入安全网：命中高风险走安全提示，不进入普通回应。
- agent 仅能读该 `user_id` 自己的数据，工具调用受 `LLM_FAST_TIMEOUT` 约束，失败有兜底文案。

### 6.31 F31 自由探索玩法扩展 + 岛屿奇遇

#### 描述
在 `ExploreMode.tsx` 内提供多种轻量治愈玩法，状态用 `localStorage` 持久化：

| 玩法 | 说明 | 关键键 |
|---|---|---|
| 心情花田（MoodGarden） | 走到花田按键种花 | `xy_garden` |
| 暮色天灯（SkyLanterns） | 夜晚放天灯，可「放飞一片」万灯齐放 + 心形 / 菊花烟花 + 地面祝福涟漪 + 八音盒旋律 | `xy_lanterns` / `xy_night` |

#### 岛屿奇遇（彩蛋系统）
走近触发的世界彩蛋，分四主题：小动物（狐狸 / 猫 / 猫头鹰 / 寄居蟹 / 归海小海龟）、夜晚魔法、海与水（夜光水母 / 听海海螺 / 退潮宝藏）、可互动仪式（祈愿铃）。每个发现经 `onDiscover` 聚合为 HUD「✨ 奇遇 N/M」，发现进度存 `localStorage`（`xy_discoveries`）。海滩另有精修道具（棕榈 / 小船 / 篝火 / 指路牌）与就近探测的可点亮石灯笼。

#### 边界
HUD 保持简洁（slim 胶囊 / 图标计数 / 单行），不喧宾夺主；玩法为氛围向，不写情绪记忆库。

### 6.32 F32 生产观测与备份

#### 描述
`backend/app/observability.py`：纯标准库 + FastAPI，不引第三方 SaaS。
- **request_id 贯穿一次请求**：复用入站 `X-Request-ID`，没有则生成；通过 ContextVar 让同一请求内任意日志自动带同一 id，便于串联排查（与 F26 的幂等 `request_id` 协同）。
- **结构化访问日志**：每个 HTTP 请求收尾落一条 access 记录（方法 / 路径 / 状态码 / 耗时 ms / 客户端 / 查询串）；纯 ASGI 中间件，WebSocket / lifespan 透传，不影响 `/ws/reflect`。
- **全局异常兜底**：未捕获异常记成带 traceback 的结构化错误日志，并在未发送响应时回干净的 500 JSON（含 request_id）。
- **可切换格式**：`LOG_FORMAT=json` 走 JSON 行日志（ELK / Loki / 云日志友好），默认 `console` 人类可读、中文不转义。
- **备份**：数据在 PostgreSQL，配套备份脚本 / 流程见仓库 `scripts/` 与 `backend/`。

## 7. 核心流程

### 7.1 普通交互流程
```text
用户输入心情（可勾选无痕）
  -> 情绪分析 Agent（LLM）：情绪 / 强度 / 摘要
  -> 安全边界 Agent（规则）：双触发判定
  -> 记忆检索 Agent（pgvector + PostgreSQL）
  -> 环境推理 Agent（纯 Python）：场景 + 岛屿状态（叠加物件）
  -> [高强度负面且非高风险] 呼吸仪式着陆
  -> 叙事表达 Agent（LLM）：叙事 + 心灵印记 + 记忆提示 -> 输出端安全复检
  -> 私房安慰话回响（如有）
  -> 岛屿回应选择卡（非高风险 / 非无痕）
  -> 保存记忆（非无痕）+ 写向量库
  -> 前端展示导演台、岛屿场景、叙事卡片、选择卡、心象地图
```

### 7.2 高风险流程
```text
用户输入高风险文本
  -> 安全边界 Agent 触发（强度阈值或关键词）
  -> 停止普通叙事（narrative="" / imprint=null / choices=[]）
  -> 返回统一求助引导（含 12356 热线）
  -> 导演台 safety 卡片显示已介入
```

### 7.3 回访流程
```text
用户再次进入
  -> 加载本地身份
  -> 读取最近记忆，计算 island_state
  -> 依次尝试：修正信 -> 离岛信件 -> 主动低语 -> 夜间守望
  -> 展示心象地图与岛屿状态摘要
  -> 用户继续倾诉 / 静默 / 写一个字 / 与岛屿对话 / 进入自由探索
```

### 7.4 对话流程（agent）
```text
用户在「说给岛屿」/ 常驻助手 / 陪伴精灵发起对话
  -> 安全网先行复检
  -> agent（DeepSeek function-calling）判断是否需要工具
       -> 需要：调用 recall_memories / read_island / list_recent 取材
       -> 不需要：直接回应
  -> 注入治愈知识库语气与边界生成回应
  -> 返回 reply/answer + tools_used（可解释）+ safety
```

  React 19 + TypeScript + Vite 前端（PWA：manifest + Service Worker + 离线兜底页）
   ├─ 3D：react-three-fiber / drei / postprocessing + three（懒加载，按性能分级；含开车与探索玩法）

## 12. 非功能需求

### 12.1 性能
- 首屏本地 3 秒内可见主要 UI；3D 懒加载，弱设备自动降级；重模型 defer 延后加载。
- Mock 模式 HTTP 1 秒内响应；真实模型 3-8 秒；WebSocket 阶段反馈避免空等。
- 最近 20 条记忆 500ms 内返回。

### 12.2 稳定性（多重降级）
| 项目 | 降级策略 |
|---|---|
| 模型失败 | 每个 provider 方法自动降级到 Mock / 模板 |
| 对话 agent 失败 | 工具调用超时 / 报错有兜底文案，不阻断页面 |
| WebSocket 失败 | 回退 HTTP，request_id 防重复落库 |
| pgvector 不可用 | 回退 PostgreSQL 最近记忆 |
| 3D / WebGL 不可用 | 回退 2D CSS 场景（开车 / 探索一并降级） |
| TTS 不可用 | 回退浏览器原生合成 |
| 音频 / 语音输入不可用 | 不阻断主流程 |
| 网络离线 | PWA 缓存外壳 + 离线兜底页，不白屏 |

### 12.3 可用性与无障碍
- 移动端可完成主要操作；软键盘不遮挡输入。
- 尊重 `prefers-reduced-motion`，提供静海模式。
- 首次进入只需昵称，不注册；提供快捷输入 / 路演种子。
- 非语言入口（静默 / 写字）降低表达门槛。

### 12.4 安全与隐私
- 匿名本地身份，不主动收集 PII。
- 双触发安全网 + 输出端复检；对话链路同样先过安全网；安全提示优先级最高。
- API Key 仅在后端环境变量；生产 CORS 只允许真实域名。
- 身份可一键删除全部后端痕迹。

### 12.5 可观测性
- 每次请求有可串联的 request_id；结构化访问日志与异常日志便于线上排查。
- `GET /api/health` 暴露 provider / 模型 / healing_kb 版本，便于核对线上与代码一致。

## 17. 总结

《心屿》把产品从「情绪输入后的单次叙事反馈」升级为「AI 驱动的情绪世界生成游戏」，并落地为一套真正可演示、可核验的系统。核心竞争力：

1. **AI 驱动游戏世界**：情绪改变岛屿场景、音乐、地图节点、成长等级与物件，而不只是生成一段文字。
2. **可解释而非黑箱**：五感信使让人看见 AI 如何一步步读懂你；该用 AI 用 AI、该用规则用规则；反思主链路确定性优先，对话陪伴才放开为真·agent。
3. **独占被忽视的人群**：静默坐岛、写一个字、夜间守望——承认「说不出也算说话」。
4. **代码级社会责任**：双端安全网、输出端防复述、隐私一键删除、去疗效化文案。
5. **持续陪伴与回访**：主动低语、离岛信件、修正信、年报、时光机、与岛屿对话，让历史心情变成可回看、可被记得的游戏轨迹。

最终答辩主张：

> **心屿不教你变得更好，它只是记得你来过、陪你长出形状。**
