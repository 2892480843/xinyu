## 1. 项目概述

### 1.1 项目背景

Tencent Cloud Hackathon: Extreme Game Dev Challenge 以 **AI-driven game creation** 为核心，鼓励高校开发者用 AI 创作新的游戏体验。《心屿》面向大学生和年轻职场人常见的焦虑、疲惫、孤独与压力问题，将 AI 文本理解、情绪记忆、动态环境推理和叙事生成结合起来，构建一个低门槛、可持续陪伴的情绪岛屿。

旧版《心屿》的核心体验是：用户输入心情，系统识别情绪，生成一段温柔叙事，并切换场景与音乐。新版将项目升级为：

> **AI 把用户情绪转化为可持续生长的游戏世界。每一次倾诉都会改变岛屿，每一次返回都能看见自己曾经走过的心理轨迹。**

### 1.2 项目愿景

打造一座会倾听、会记忆、会生长的 AI 心灵岛屿，让用户在压力和孤独时获得一个低门槛、无羞耻感、可持续回访的情绪出口。

### 1.3 项目一句话

《心屿》不是普通聊天机器人，而是一个由多智能体驱动的情绪世界生成游戏：AI 会理解用户情绪、检索过往记忆、推理岛屿状态，并生成叙事、场景、音乐与心象地图。

### 1.4 项目类型

| 维度 | 内容 |
|---|---|
| 赛事方向 | AI 驱动游戏创作 |
| 游戏类型 | 治愈叙事游戏、情绪陪伴游戏、轻量交互式体验 |
| 社会价值方向 | 青年心理压力缓解、低门槛情绪表达、安全风险提示 |
| 目标平台 | Web 端优先，兼容移动端浏览器 |
| 核心玩法 | 倾诉心情、观看岛屿变化、收集心灵印记、回看情绪轨迹 |

## 2. 用户与场景

### 2.1 目标用户

| 用户类型 | 特征 | 核心诉求 |
|---|---|---|
| 高校学生 | 学业、竞赛、就业、社交压力较高 | 想找一个低压力的情绪出口 |
| 年轻职场人 | 加班、转型、孤独感、成就焦虑明显 | 希望被理解，也希望看见自己慢慢变好 |
| 黑客松评委 | 关注 AI 创新、游戏性、技术完整度、社会价值 | 希望看到 AI 不只是生成文本，而是驱动游戏世界 |

### 2.2 典型使用场景

| 场景 | 用户行为 | 系统回应 |
|---|---|---|
| 焦虑倾诉 | 用户输入“明天答辩，我很紧张” | AI 识别焦虑，岛屿出现雾气，叙事引导用户放慢呼吸 |
| 疲惫回访 | 用户多次记录“很累” | 岛屿夜色加深，但灯塔逐渐亮起，提示用户曾经也走过低谷 |
| 开心记录 | 用户输入“今天拿奖了，很开心” | 岛屿长出花，音乐变明亮，保存一枚心灵印记 |
| 高风险表达 | 用户输入强烈绝望或自伤相关内容 | 暂停普通叙事，显示安全提示与求助建议 |
| 答辩演示 | 演示者连续输入 3 段不同情绪 | 多智能体导演台逐步点亮，心象地图生成情绪轨迹 |

## 3. 产品定位与核心创新

### 3.1 新版定位

旧版定位：

> AI 情感陪伴叙事游戏。

新版定位：

> **AI 驱动的情绪世界生成游戏。**

新版不只生成一段安慰文本，而是把用户情绪转化为三层游戏结果：

| 层级 | 说明 | 用户感知 |
|---|---|---|
| 即时回应层 | 识别情绪并生成治愈叙事 | “它听懂了我现在的心情” |
| 记忆连续层 | 检索历史情绪并生成回访感 | “它记得我之前发生过什么” |
| 世界生成层 | 根据情绪历史改变岛屿状态 | “我的心情真的改变了这座岛” |

### 3.2 核心创新点

| 创新点 | 说明 | 比赛价值 |
|---|---|---|
| 心象岛屿生长系统 | 情绪记忆会转化为岛屿元素，例如雾、灯塔、花、星光、雨痕、贝壳 | 强化游戏性和视觉表现 |
| 多智能体导演台 | 展示情绪分析、记忆检索、环境推理、叙事表达等 Agent 的协作过程 | 证明 AI 系统架构，而不是简单调用模型 |
| 情绪记忆驱动世界状态 | 最近情绪趋势会影响岛屿成长等级、天气记忆、主导元素 | 体现持续陪伴和技术壁垒 |
| 心象地图 | 历史情绪不再只是列表，而是岛上的时间线节点 | 增强可玩性和回访动力 |
| 安全边界机制 | 对高风险情绪暂停普通叙事，进入安全提示 | 体现社会责任与产品边界 |

## 4. 术语说明

| 术语 | 解释 |
|---|---|
| AI Agent（AI 智能体） | 负责完成某一类任务的 AI 模块，例如专门判断情绪、专门生成故事、专门决定场景 |
| 多智能体协作 | 多个 AI Agent 分工处理同一次用户输入，最后组合成完整游戏反馈 |
| 情绪记忆 | 系统保存的用户历史心情记录，包括文本、情绪标签、强度、叙事摘要和时间 |
| 向量记忆 | 把文本转成数学向量，用于检索“语义上相似”的历史记忆 |
| 心象岛屿 | 用户情绪在游戏世界中的可视化表现，岛屿会随记忆变化 |
| 心象地图 | 把历史情绪节点显示成岛屿上的轨迹或标记 |
| 心灵印记 | 每次交互后生成的一句短诗或语录，可保存、收藏或分享 |
| 安全提示 | 当系统检测到高风险表达时，停止普通叙事并提示用户联系可信任的人或专业资源 |

## 6. 详细功能需求

### 6.1 F1 情绪入口

#### 6.1.1 功能描述

用户进入《心屿》后，通过文字或语音输入当下心情。输入不要求专业表达，可以是短句、长段倾诉或口语化描述。

#### 6.1.2 输入方式

| 输入方式 | 优先级 | 说明 |
|---|---|---|
| 文字输入 | P0 | 核心输入方式，支持多行文本 |
| 快捷情绪短句 | P0 | 提供 3-4 个示例输入，便于演示 |
| 浏览器语音输入 | P1 | 使用浏览器原生 SpeechRecognition，无法使用时降级 |

#### 6.1.3 输入限制

| 项目 | 要求 |
|---|---|
| 最大长度 | 2000 字以内 |
| 空文本 | 禁止提交 |
| 用户身份 | 使用本地匿名 `user_id`，不要求注册登录 |
| PII 提醒 | 不鼓励用户填写真实姓名、手机号、学号、邮箱等个人身份信息 |

#### 6.1.4 验收标准

- 用户可以输入文本并提交。
- 提交后进入 AI 处理流程。
- 空文本不能提交。
- 后端能返回明确错误信息。

---

### 6.2 F2 多智能体导演台

#### 6.2.1 功能描述

多智能体导演台用于展示 AI 如何把一次用户输入转化成游戏体验。它不只是加载动画，而是比赛答辩中的技术可解释窗口。

#### 6.2.2 Agent 设计

| Agent | 中文名 | 职责 | 输出 |
|---|---|---|---|
| Emotion Agent | 情绪分析 Agent | 判断情绪类别、强度和摘要 | `emotion`、`intensity`、`summary` |
| Memory Agent | 记忆检索 Agent | 检索最近或相似历史情绪 | 记忆数量、相似情绪提示 |
| Environment Agent | 环境推理 Agent | 决定岛屿时间、天气、色彩、音乐和元素 | `scene`、`island_state` |
| Narrative Agent | 叙事表达 Agent | 生成治愈叙事和心灵印记 | `narrative`、`imprint` |
| Safety Agent | 安全边界 Agent | 判断是否触发高风险提示 | `safety` |

#### 6.2.3 前端展示

导演台以 4-5 张轻量卡片展示，每张卡片包含：

| 字段 | 示例 |
|---|---|
| Agent 名称 | 情绪分析 Agent |
| 状态 | 等待中 / 处理中 / 已完成 |
| 输出摘要 | 识别为疲惫，强度 0.72 |
| 图标或状态灯 | 圆点、进度线、轻量动画 |

#### 6.2.4 WebSocket 事件

建议新增或统一以下事件：

```json
{
  "event": "agent",
  "agent": "emotion",
  "label": "情绪分析 Agent",
  "status": "done",
  "output": "识别为疲惫，强度 0.72"
}
```

#### 6.2.5 验收标准

- 用户提交后，导演台按阶段展示 Agent 处理过程。
- 每个 Agent 至少有一条可读输出摘要。
- 普通 HTTP 回退时，也能展示最终 `agent_trace`。
- 高风险场景下，Safety Agent 必须显示已触发安全边界。

### 6.3 F3 心象岛屿生长系统

#### 6.3.1 功能描述

心象岛屿生长系统根据当前输入和历史记忆生成一个 `island_state`。它是新版《心屿》的核心创新：用户情绪不只是被记录，而是会改变岛屿。

#### 6.3.2 岛屿状态字段

```json
{
  "dominant_emotion": "tired",
  "trend": "recovering",
  "growth_level": 3,
  "features": ["stars", "lighthouse", "quiet_tree"],
  "weather_memory": "recent_rain",
  "summary": "你的岛屿最近夜色较深，但灯塔已经亮起来了。"
}
```

#### 6.3.3 字段说明

| 字段 | 类型 | 说明 |
|---|---|---|
| `dominant_emotion` | string | 最近记忆中最主要的情绪 |
| `trend` | string | 情绪趋势，例如 `recovering`、`stable`、`stormy`、`brightening` |
| `growth_level` | number | 岛屿成长等级，建议 1-5 |
| `features` | string[] | 岛屿元素，例如灯塔、星星、花、雾、贝壳 |
| `weather_memory` | string | 近期天气记忆，例如 recent_rain、clearing_sky |
| `summary` | string | 给用户看的岛屿状态一句话 |

#### 6.3.4 岛屿元素映射

| 情绪 | 岛屿元素 | 视觉含义 |
|---|---|---|
| `sad` 难过 | 雨痕、灯塔、潮水 | 情绪被看见，岸仍在 |
| `anxious` 焦虑 | 雾、礁石、缓慢风线 | 思绪混乱但可慢慢散开 |
| `tired` 疲惫 | 星光、吊床、低速海浪 | 需要休息和停靠 |
| `lonely` 孤独 | 单棵树、远船、月光 | 一个人也被岛屿记得 |
| `calm` 平静 | 帆船、清水、晨光 | 内在稳定 |
| `happy` 愉悦 | 花、蝴蝶、阳光 | 喜悦被收藏 |
| `angry` 愤怒 | 崖壁、风暴、浪花 | 边界感和能量释放 |
| `helpless` 无助 | 微光、静水、避风处 | 先停靠，不独自穿过黑夜 |

#### 6.3.5 成长等级规则

| 等级 | 条件建议 | 表现 |
|---|---|---|
| 1 | 0-1 条记忆 | 初始岛屿，元素较少 |
| 2 | 2-4 条记忆 | 出现基础地貌和天气记忆 |
| 3 | 5-9 条记忆 | 出现灯塔、树、花等长期元素 |
| 4 | 10-19 条记忆 | 心象地图更完整，出现趋势摘要 |
| 5 | 20 条及以上记忆 | 岛屿形成稳定风格，可展示长期轨迹 |

#### 6.3.6 趋势判断规则

| 趋势 | 判断逻辑建议 | 展示文案方向 |
|---|---|---|
| `recovering` | 近期强度下降，负面情绪减少 | “云层正在散开” |
| `stormy` | 高强度负面情绪连续出现 | “岛上风浪较大，需要更多停靠” |
| `brightening` | 愉悦或平静情绪增加 | “岛屿正在变亮” |
| `stable` | 情绪强度波动较小 | “岛屿保持平稳潮汐” |
| `mixed` | 多种情绪交替明显 | “岛屿有多种天气交错” |

#### 6.3.7 验收标准

- 每次普通情绪提交后返回 `island_state`。
- 历史记忆数量变化时，`growth_level` 有合理变化。
- 不同主导情绪返回不同 `features`。
- 前端能根据 `features` 展示至少 3 类可见变化。

### 6.4 F4 AI 治愈叙事

#### 6.4.1 功能描述

系统根据情绪、强度、场景意象和历史记忆生成 50-120 字中文叙事。叙事风格应温柔、克制、有画面感，不做医疗诊断，不承诺治疗效果。

#### 6.4.2 叙事原则

| 原则 | 要求 |
|---|---|
| 温柔克制 | 不使用夸张鸡汤，不强行积极 |
| 画面化 | 尽量使用岛屿、海浪、光、雨、风等意象 |
| 有边界 | 不说“你一定会好”“我能治愈你” |
| 有连续感 | 若有历史记忆，应自然体现“岛屿记得你” |
| 安全优先 | 高风险时不输出普通故事 |

#### 6.4.3 输出示例

```json
{
  "narrative": "夜深了，岛上的风很轻。你躺在吊床里，星星替你守着今晚。今天已经很努力了，那些做不好的事，先搁在岸边吧，岛屿会替你看着。",
  "imprint": "今晚先把自己交给星光，明天的海会替你重新托起帆。",
  "memory_hint": "岛屿记得你上次也带着疲惫来过，但你依然走到了今天。"
}
```

#### 6.4.4 验收标准

- 普通输入能生成叙事、心灵印记和记忆提示。
- 叙事长度控制在 50-120 字为主。
- 输出不得包含医疗诊断或治疗承诺。
- 真实模型失败时可降级到本地 Mock 模板。

### 6.5 F5 情绪记忆系统

#### 6.5.1 功能描述

系统保存用户每次情绪记录，并在后续交互中用于生成连续陪伴感和心象岛屿状态。

#### 6.5.2 存储字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | number | 记忆 ID |
| `user_id` | string | 本地匿名用户 ID |
| `text` | string | 用户原始输入 |
| `emotion` | string | 情绪标签 |
| `intensity` | number | 情绪强度，0-1 |
| `summary` | string | 情绪摘要 |
| `narrative` | string | 本次生成叙事 |
| `imprint` | string | 心灵印记，P1 可落库 |
| `created_at` | string | 创建时间 |

#### 6.5.3 检索机制

| 检索方式 | 优先级 | 说明 |
|---|---|---|
| 最近记忆 | P0 | 取最近 3-20 条记录，用于趋势和地图 |
| 向量相似记忆 | P1 | 使用 ChromaDB 检索语义相似情绪 |
| 情绪聚合统计 | P0 | 统计主导情绪、平均强度、负面比例 |

#### 6.5.4 隐私边界

- 不要求用户注册。
- 不保存手机号、学号、邮箱、真实姓名。
- `user_id` 仅用于本地 Demo 隔离。
- 用户应能清除本地身份。
- 后端应提供按 `user_id` 删除记忆的能力。

#### 6.5.5 验收标准

- 不同 `user_id` 的记忆互相隔离。
- 记忆保存后能在心象地图中展示。
- 删除指定用户记忆后，历史列表和地图清空。

### 6.6 F6 心象地图

#### 6.6.1 功能描述

心象地图将历史情绪记录从普通列表升级为游戏化轨迹。每条记忆都是岛上的一个节点，可以表现为光点、贝壳、星星、花、雨滴或脚印。

#### 6.6.2 地图展示形态

MVP 阶段建议采用“轻量地图 + 时间线”组合：

| 展示方式 | 说明 |
|---|---|
| 岛屿节点 | 最近 8-12 条记忆以节点形式分布在小地图上 |
| 情绪颜色 | 不同情绪对应不同颜色或图标 |
| 强度大小 | 情绪强度越高，节点越明显 |
| 时间线 | 展示创建时间、情绪标签和输入摘要 |
| 点击查看 | 点击节点可查看原始输入摘要和叙事片段 |

#### 6.6.3 节点映射建议

| 情绪 | 节点形态 |
|---|---|
| 难过 | 雨滴 |
| 焦虑 | 雾点 |
| 疲惫 | 星点 |
| 孤独 | 月光贝壳 |
| 平静 | 小帆 |
| 愉悦 | 花 |
| 愤怒 | 火色浪花 |
| 无助 | 微光 |

#### 6.6.4 验收标准

- 地图能展示最近记忆节点。
- 节点能体现情绪类型和强度。
- 地图为空时显示温柔空状态。
- 移动端不遮挡主输入和叙事卡片。

### 6.7 F7 多模态环境反馈

#### 6.7.1 功能描述

系统根据当前情绪和岛屿状态切换场景、音乐和动画，使用户感到 AI 正在生成一个与自己有关的世界。

#### 6.7.2 视觉反馈

| 类型 | P0 要求 | P1 扩展 |
|---|---|---|
| 场景图 | 使用 8 种情绪 x 3 强度的本地预设图 | 引入 AI 图像生成或云端异步生成 |
| 动画 | 背景淡入淡出、海浪、雾、星光微动 | 根据 `island_state.features` 叠加元素 |
| 岛屿元素 | 灯塔、花、星点、雾层等基础元素 | 可收集、可点击、可解释来源 |

#### 6.7.3 音频反馈

| 情绪 | 音乐建议 |
|---|---|
| 难过 | soft piano |
| 焦虑 | ambient drone |
| 疲惫 | soft piano |
| 孤独 | ambient drone |
| 平静 | lofi |
| 愉悦 | bright acoustic |
| 愤怒 | low strings |
| 无助 | soft piano |

#### 6.7.4 验收标准

- 不同情绪能切换不同场景和音乐。
- 场景切换有平滑过渡。
- 音乐默认不强制自动播放，符合浏览器策略。
- 视觉效果不影响文字可读性。

### 6.8 F8 心灵印记

#### 6.8.1 功能描述

每次普通交互结束后，系统生成一句 20-60 字的中文心灵印记。印记可以保存为 PNG，也可以在后续版本沉淀为岛上的收藏物。

#### 6.8.2 印记要求

| 项目 | 要求 |
|---|---|
| 长度 | 20-60 字 |
| 风格 | 短诗、语录、温柔克制 |
| 内容 | 呼应本次情绪和岛屿意象 |
| 禁止 | 口号化、诊断化、承诺治疗 |

#### 6.8.3 验收标准

- 普通输入后生成印记。
- 高风险输入时印记为 `null`。
- 用户可以保存印记为 PNG。

### 6.9 F9 情绪恢复趋势

#### 6.9.1 功能描述

系统根据近期情绪强度和情绪类型，生成一个轻量趋势判断，帮助用户看到“自己不是一直停在原地”。

#### 6.9.2 趋势展示

| 趋势 | 表达方式 |
|---|---|
| 正在恢复 | 云层散开、灯塔变亮 |
| 波动明显 | 潮汐起伏、天气交错 |
| 持续高压 | 风浪较大、提示休息和求助 |
| 保持平稳 | 海面平静、帆船缓行 |

#### 6.9.3 产品边界

趋势只作为游戏化回顾，不作为心理测评、医学判断或治疗建议。

### 6.10 F10 路演模式

#### 6.10.1 功能描述

为了黑客松答辩稳定演示，系统提供路演模式。路演模式可以一键填入示例输入，并按预设顺序展示完整流程。

#### 6.10.2 示例流程

| 步骤 | 示例输入 | 展示重点 |
|---|---|---|
| 1 | “我明天答辩，有点焦虑，怕自己讲不好。” | 情绪分析 Agent 与雾气场景 |
| 2 | “昨晚加班到很晚，今天真的很累。” | 记忆检索 Agent 与疲惫场景 |
| 3 | “但今天也收到一个好消息，感觉有一点开心。” | 岛屿趋势从阴沉转向变亮 |
| 4 | “我真的彻底撑不下去了，感觉没有希望。” | 安全边界 Agent 触发 |

#### 6.10.3 验收标准

- 演示者可以快速填充示例文本。
- 可以稳定展示 Agent 导演台、岛屿变化和心象地图。
- 高风险示例能稳定触发安全提示。

### 6.11 F11 风险预警

#### 6.11.1 功能描述

当用户表达强烈绝望、自伤、他伤或极端无助倾向时，系统必须暂停普通治愈叙事，显示安全提示。

#### 6.11.2 触发条件

| 条件 | 说明 |
|---|---|
| 情绪类型 | `sad`、`angry`、`helpless` 等高风险情绪 |
| 情绪强度 | 强度达到阈值，例如 `>= 0.85` |
| 关键词 | 包含强烈绝望、自伤、他伤、撑不下去等表达 |

#### 6.11.3 安全提示原则

| 原则 | 要求 |
|---|---|
| 不诊断 | 不判断用户患有什么疾病 |
| 不承诺 | 不承诺平台可以解决危机 |
| 鼓励求助 | 建议联系可信任的人、专业机构或紧急电话 |
| 本地化 | 根据部署地区提供合适热线或紧急资源 |

#### 6.11.4 验收标准

- 高风险输入不生成普通叙事。
- 返回 `safety.triggered = true`。
- 前端显示安全提示页面。
- 安全提示文案清晰、克制、可操作。

## 7. 核心流程

### 7.1 普通交互流程

```text
用户输入心情
  -> Safety Agent 初步判断风险
  -> Emotion Agent 分析情绪和强度
  -> Memory Agent 检索历史记忆
  -> Island State Engine 聚合岛屿状态
  -> Environment Agent 推理场景、天气、音乐、元素
  -> Narrative Agent 生成叙事、心灵印记、记忆提示
  -> 保存情绪记忆
  -> 前端展示导演台、岛屿场景、叙事卡片、心象地图
```

### 7.2 高风险流程

```text
用户输入高风险文本
  -> Safety Agent 触发安全边界
  -> 停止普通叙事生成
  -> 保存必要的风险记忆摘要
  -> 前端显示安全提示
  -> 导演台展示 Safety Agent 已介入
```

### 7.3 回访流程

```text
用户再次进入
  -> 加载本地身份
  -> 读取最近记忆
  -> 生成当前 island_state
  -> 展示心象地图和岛屿状态摘要
  -> 用户继续倾诉
```

## 9. API 需求

### 9.1 健康检查

```http
GET /api/health
```

返回系统状态、模型配置和支持的情绪列表。

### 9.2 情绪反思接口

```http
POST /api/reflect
Content-Type: application/json
```

请求：

```json
{
  "user_id": "local-demo-user",
  "text": "我今天有点焦虑。"
}
```

响应：返回 `ReflectResponse`。

### 9.3 WebSocket 流式接口

```http
WS /ws/reflect
```

客户端发送：

```json
{
  "user_id": "local-demo-user",
  "text": "我今天有点焦虑。"
}
```

服务端事件：

| 事件 | 说明 |
|---|---|
| `started` | 开始处理 |
| `agent` | 推送单个 Agent 进度和摘要 |
| `emotion` | 推送情绪分析结果 |
| `scene` | 推送场景结果 |
| `island_state` | 推送岛屿状态 |
| `narrative` | 推送叙事和印记 |
| `memory` | 推送保存后的记忆 |
| `done` | 推送最终完整响应 |
| `error` | 推送错误信息 |

### 9.4 记忆列表接口

```http
GET /api/memories?user_id=local-demo-user&limit=20
```

返回指定用户最近记忆。

### 9.5 岛屿状态接口

可选新增：

```http
GET /api/island-state?user_id=local-demo-user
```

用途：用户首次进入页面时，无需提交新心情，也能看到当前岛屿状态。

### 9.6 删除记忆接口或脚本

P0 可使用本地脚本，P1 可提供接口：

```http
DELETE /api/memories?user_id=local-demo-user
```

## 11. 技术架构

### 11.1 总体架构

```text
[用户浏览器]
  -> React + TypeScript + Vite 前端
  -> HTTP / WebSocket
  -> FastAPI 后端
  -> 多智能体编排层
      -> Safety Agent
      -> Emotion Agent
      -> Memory Agent
      -> Environment Agent
      -> Narrative Agent
  -> 记忆层
      -> SQLite 作为主存储
      -> ChromaDB 作为可选向量检索
  -> LLM Provider
      -> Mock 离线模式
      -> OpenAI 兼容接口 / 腾讯云模型适配
```

### 11.2 推荐技术栈

| 层 | 技术 | 说明 |
|---|---|---|
| 前端 | React + TypeScript + Vite | 快速开发交互式 Web 游戏 |
| 动画 | Framer Motion | 场景过渡、卡片动画、导演台状态变化 |
| 样式 | Tailwind CSS | 快速实现响应式界面 |
| 后端 | FastAPI | HTTP 与 WebSocket 支持成熟 |
| 存储 | SQLite | Demo 阶段稳定、简单、可复现 |
| 向量库 | ChromaDB | 可选语义记忆检索 |
| 大模型 | OpenAI 兼容接口 / 腾讯云模型 | 便于后续迁移到赛事云资源 |
| 部署 | Docker + 腾讯云服务器 | 便于答辩展示和线上访问 |

### 11.3 云服务适配方向

| 能力 | 腾讯云适配建议 |
|---|---|
| Web 部署 | 腾讯云轻量应用服务器、CloudBase 或容器服务 |
| 模型调用 | 腾讯云大模型或兼容 API 网关 |
| 对象存储 | COS 保存印记图片、场景资源或导出演示素材 |
| 日志监控 | 腾讯云日志服务用于记录接口性能和错误 |
| 安全合规 | HTTPS、CORS 白名单、敏感信息脱敏 |

## 15. 答辩策略

### 15.1 核心讲法

> 《心屿》的核心创新是用 AI 把用户情绪转化为可持续生长的游戏世界。多智能体系统负责理解情绪、检索记忆、推理环境并生成叙事，最终让每一次倾诉都成为岛屿的一部分。

### 15.2 三个答辩亮点

| 亮点 | 讲法 |
|---|---|
| AI 驱动游戏世界 | AI 输出不只是文字，而是影响天气、地貌、音乐、地图节点 |
| 多智能体可解释流程 | 导演台展示 AI 从理解到生成的全过程 |
| 社会价值闭环 | 面向大学生压力和孤独，提供低门槛表达入口，同时设置高风险安全边界 |

### 15.3 演示脚本

1. 打开首页，输入昵称进入岛屿。
2. 输入“我明天答辩，有点焦虑，怕自己讲不好。”
3. 展示导演台：情绪分析、记忆检索、环境推理、叙事表达依次完成。
4. 展示焦虑场景：雾气、低速环境音乐、叙事卡片。
5. 输入“昨晚加班到很晚，今天真的很累。”
6. 展示岛屿状态变化：夜色、星光、灯塔或休息元素。
7. 打开心象地图，展示两条记忆节点。
8. 输入“但今天收到好消息，感觉有一点开心。”
9. 展示岛屿趋势变亮，地图新增愉悦节点。
10. 输入高风险示例，展示安全提示，强调产品边界。

### 15.4 评委可能提问与回答

| 问题 | 回答 |
|---|---|
| 这和普通聊天机器人有什么区别？ | 普通聊天机器人只生成文本，《心屿》会把情绪转化为游戏世界状态，包括场景、音乐、地图和岛屿成长。 |
| 多智能体是真的有价值吗？ | 我们把复杂生成任务拆成情绪、记忆、环境、叙事和安全边界，每个模块可独立优化，也能在导演台中解释 AI 决策过程。 |
| 如何体现社会价值？ | 它降低了情绪表达门槛，适合不愿直接寻求咨询的年轻人先表达压力；同时高风险表达会触发安全提示，不替代专业服务。 |
| 隐私如何保护？ | Demo 阶段不要求注册，不收集真实身份，只用本地匿名 ID 隔离记忆，并支持删除。 |
| 模型失败怎么办？ | 系统保留 Mock 模式和模板降级，保证黑客松演示稳定。 |

## 16. 风险与应对

| 风险 | 影响 | 应对 |
|---|---|---|
| 实时模型响应慢 | 影响演示节奏 | WebSocket 阶段反馈 + Mock 降级 |
| 图像生成不稳定 | 演示翻车 | P0 使用本地预设场景，P1 再接入 AI 图像 |
| 多智能体实现成本过高 | 开发超时 | P0 先用现有流程包装 Agent Trace |
| 心理安全争议 | 产品风险 | 明确非医疗定位，高风险时停止普通叙事 |
| 功能过多导致分散 | 答辩重点不清 | 主线只讲“情绪记忆驱动岛屿生长 + 多智能体导演台” |
| 移动端遮挡 | 体验下降 | 心象地图可折叠，导演台只在加载和结果页轻量展示 |

## 18. 版本边界

### 18.1 P0 必须完成

- 情绪输入
- 情绪分析
- AI 叙事
- 风险提示
- 记忆保存
- 岛屿状态生成
- 多智能体导演台
- 心象地图轻量版
- 场景和音乐反馈

### 18.2 P0 不做

- 真实账号系统
- 复杂社交功能
- 医疗诊断或心理治疗建议
- 强依赖实时图像生成
- 付费系统
- 大规模多人在线

### 18.3 P1 可选

- 路演模式
- 印记收藏墙
- 情绪趋势图
- 腾讯云部署
- 云端对象存储

## 19. 总结

新版《心屿》围绕 Tencent Cloud Hackathon: Extreme Game Dev Challenge 的 AI 游戏创作方向，将产品从“情绪输入后的单次叙事反馈”升级为“AI 驱动的情绪世界生成游戏”。

项目的核心竞争力是：

1. **AI 驱动游戏世界**：用户情绪会改变岛屿，而不只是生成一段文字。
2. **多智能体可解释生成**：导演台让评委看到 AI 如何理解、记忆、推理和表达。
3. **情绪记忆形成长期陪伴**：心象地图让历史心情变成可回看的游戏轨迹。
4. **社会价值明确**：面向高校青年压力和孤独问题，同时保持安全边界。

最终答辩主张：

> **每一次倾诉，都会成为岛屿的一部分。**
