# 心屿 · 路演物料

> 面向：腾讯云黑客松「AI CAN DO IT 极限游戏开发挑战赛」答辩 / 路演
> 用途：现场口播底稿 + 逐 beat 演示脚本 + 防翻车清单 + Q&A
> 原则：只写代码里真实存在的东西；措辞克制，不夸大，不做疗效承诺。

---

## 1. 一句话定位 + slogan

**定位**：心屿是一座会回应你的「心象岛屿」——你把当下的心情留给它，它用情绪分析读懂你、用克制的中文叙事陪你，并把这一刻沉淀成可被记得、可被回望、也可被删除的情绪记忆。

**Slogan**：

> **心屿不教你变得更好，它只是记得你来过、陪你长出形状。**

它不是工具，不是教练，不是咨询师。它是一个允许你「什么都做不好、什么都说不出」时也能来坐一会儿的地方。

---

## 2. 赛道卡位

心屿对准社会价值「小红花」赛题（含适老化 / 独居老人方向），用三层逻辑卡住位置：游戏性 × AI 中心性 × 社会价值，缺一不可。

### 第一层：人群独占——我们服务别人不愿意服务的那段时间

市面上的情绪 / 治愈类产品几乎都默认用户「愿意表达、能够表达、想要变好」。心屿独占的是相反的那段时间：

- 想来，但说不出话——我们有「静默坐岛」：不打字，岛屿陪坐 N 秒，留下一枚静默贝壳作为「我来过」的证据（`POST /api/silent/companion`，不写记忆库，承认沉默是合法情绪状态）。
- 写不动一句完整的话——我们有「写一个字给岛屿」：用户描红一个心境字，岛屿结合字义 + 书写动力学（笔速 / 停顿 / 抖动）读出情绪（`POST /api/glyph`）。
- 独居 / 适老化场景——低交互门槛（一句话、一个字、甚至只是坐着），离岛超过一段时间岛屿会主动写一封「我替你看着」的离岛信件（`/api/island/welcome-back`），降低「没人惦记」的孤独感。

### 第二层：安全即社会价值——我们有代码级的硬证据

公益赛题最怕「嘴上有温度、代码里没护栏」。心屿的社会价值不是文案，是可在源码里逐行核验的安全机制：

- **双触发安全网**（`safety_service.py`）：情绪强度阈值（`sad/angry/helpless/anxious` 且强度 ≥ 0.85）**或**关键词黑名单命中，任一即触发。关键词是独立于强度的第二条线——专门修复「平静措辞的自伤表达因强度低而漏过」的漏洞。
- **触发即让位**：一旦触发，普通治愈叙事被整段暂停（`narrative=""`、`imprint=None`、不给选择卡），改为统一的求助引导文案，引导联系可信任的人或拨打热线。
- **输出端复检防复述**：`memory_is_high_risk` 在岛屿低语 / 年报 / 修正信等会「回看历史」的功能里，对历史记忆原文做二次复检，把高风险记忆从素材里剔除，防止 LLM 把用户过去的自伤原文复述出来。

这三点是「面对脆弱用户的工程克制」的代码级证据，也是公益赛道里最有说服力的部分。

### 第三层：承认边界——衔接专业，而非取代专业

心屿在产品里反复、显式地声明自己的边界：「不是心理咨询或医疗服务，不诊断，不承诺疗效，只能陪伴片刻」。触发安全边界时主动把人交回给可信任的人与专业热线。我们不假装能治愈，这种诚实本身就是社会价值的一部分。

## 3. 「多智能体」诚实化话术

### 对外统一口径

不用 multi-agent / 自治智能体 / agent 集群 这类词。对外一律称：

> **可解释的情感 AI 编排管道**，或更口语的——**「岛屿用五感聆听你」**。

代码里 `agent_trace` 的五张卡片（情绪 / 记忆 / 环境 / 叙事 / 安全）对用户和评委的真实价值是**可解释性**：让人看见「岛屿是怎么一步步读懂你的」，而不是宣称它们是五个会自主通信的智能体。

### 口播定调句（务必原样说出）

> 「我们刻意没做黑箱 agent——该用 AI 用 AI，该用规则用规则。因为面对脆弱用户，**可解释和确定性比『看起来很 AI』更重要**。」

### 架构图（口播配图，可直接投屏）

```
用户输入
  → [情绪分析  混元 LLM]              ← AI
  → [安全边界  规则引擎+归一化]        ← 确定性（刻意）
  → [记忆检索  向量/SQLite]            ← 确定性
  → [环境/岛屿状态  纯 Python 状态机]  ← 确定性（断网可跑）
  → [叙事表达  混元 LLM (+情感 TTS)]   ← AI
  → [输出端安全复检  规则引擎]          ← 确定性（双端安全网）
  → 沉淀情绪记忆
```

**与真实实现的对应关系**（被追问时可逐条印证）：

- 整条 `_reflect_pipeline` 是一条**线性管道**（`main.py`）。
- 真正调用 LLM 的只有**两步**：情绪分析（`emotion_service` → 混元）与叙事表达（`narrative_service` → 混元）。
- 其余阶段都是**确定性函数**：安全边界是纯规则（`safety_service`）、场景与岛屿成长是纯 Python 状态机（`scene_map` / `island_state_service`）、记忆检索是向量库 + SQLite。
- TTS 用浏览器 `speechSynthesis`，无法用时前端降级为不可用，不影响主流程。

### 被评委追问「你的 agent 之间怎么通信」时怎么答

坦诚回答，把「没有自治通信」讲成有意的工程选择，而不是缺陷：

> 「老实说，它们之间**没有自治通信**——这是我们有意的工程选择。整条管道是一条线性流水线：上一阶段的确定性输出，作为下一阶段的输入，仅此而已。我们没有让多个智能体互相对话、互相博弈，因为面对的是情绪脆弱的用户，**不可预测的自治协作恰恰是我们要规避的风险**。
>
> 这条管道里**唯一一处多步 AI 协作，是输出端的安全复检**：在岛屿主动回看历史的功能（低语 / 年报 / 修正信）里，是『生成 → 规则批判 → 必要时降级重写』——比如复检发现某条历史记忆是高风险，就把它从素材里剔除，低语降级成一句最克制的环境化通用文案。这一处多步协作完全是**为安全而做**，不是为了『显得很 agent』。」

## 4. 5 分钟路演逐 beat 脚本

总时长约 280 秒（留 ~20 秒缓冲）。

| # | Beat | 秒 | 现场操作 | 评委此刻感受 | 埋的记忆点 |
|---|---|---|---|---|---|
| ① | 反差钩子 + 静默坐岛 | 40s | 不说产品、先抛反差句（见 M2 逐字稿）。随即在岛屿界面**不打字**，点静默坐岛，让岛屿陪坐，留下一枚静默贝壳 | 「这跟我见过的 AI 都不一样，它居然允许我不说话」 | 静默贝壳 = 「说不出也算说话」 |
| ② | 一次倾诉 + 五感信使 + 生长瞬间 | 80s | 输入一句真实心情（如疲惫文本）。讲解时强调：情绪 / 记忆 / 环境 / 叙事 / 安全五张卡片**逐个抵达**（信使逐个点亮），最后叙事卡与心灵印记落地，岛屿场景随情绪切换 | 「分析过程是看得见的，不是黑箱；画面和文字都在回应我」 | 五感信使 = 可解释；岛屿在「长出形状」 |
| ③ | AI 角色澄清 + 架构图 | 25s | 投屏第 3 章架构图，说定调句：该用 AI 用 AI、该用规则用规则 | 「他们想清楚了 AI 该放在哪、不该放在哪」 | 可解释 + 确定性 > 看起来很 AI |
| ④ | 时光机 · 一键回望情绪峰值 | 80s | 触发岛屿年报 / 修正信 / 离岛信件，展示「岛屿读懂了你的一段情绪轨迹」。**口播必须明示：这是预置示范数据**；强调轨迹有起伏、文案不评判不预测、不替代专业判断 | 「它真的记得这个人，而且记得很克制」 | 时光机 = 记得你来过；去疗效化 |
| ⑤ | 安全底线 + 求助软着陆 + 删除记忆 | 35s | 输入一条高风险文本，展示普通叙事被整段暂停、改为求助引导；再展示记忆可删除（隐私可控） | 「最该负责的地方，他们真的写进了代码」 | 安全是硬证据；隐私可删除 |
| ⑥ | 卡位 + 金句收束 | 20s | 一句话收束赛道卡位，落回 slogan | 「记住了这个产品和这句话」 | 「记得你来过、陪你长出形状」 |

> 提示：beat ② 与 beat ④ 是情绪高点，务必留足时间让画面与文字「落地」，不要赶。beat ③ 是理性锚点，必须紧凑。

## 5. 开场定位句（M2）逐字稿

开场不讲功能、不讲技术，先用一句反差句立住定位：

> 「市面上的 AI 都在教你变得更好，心屿是唯一一个，在你什么都说不出来的时候，允许你只是来坐一会儿。」

说完这句，**不要继续讲**——直接转身演示「静默坐岛」：

- 在岛屿界面**不打一个字**，点进静默陪伴；
- 岛屿安静地陪坐一小段时间，结束后**留下一枚静默贝壳**作为「我来过」的凭证；
- 这一步对应 `POST /api/silent/companion`：不写记忆库（没有倾诉文本），只留一枚贝壳。

用「先抛反差句、再用沉默演示」的方式，让评委在前 40 秒就记住：心屿的差异点不是更聪明的 AI，而是**它接住了别人不接的那段沉默**。

## 6. 伦理红线清单

这些是必须守住、且现场要能自证的红线：

1. **去疗效化**：情绪轨迹要有起伏、不修饰成「一路变好」；所有岛屿主动文案（叙事 / 印记 / 年报 / 修正信 / 低语）统一坚持「不评判、不预测、不替代专业判断」。产品文案明确写明「不是心理咨询或医疗服务，不诊断，不承诺疗效」。
2. **求助永不被呼吸 / 氛围阻挡**：安全边界一旦触发，普通叙事被整段暂停，求助引导优先级最高，不会被任何「呼吸 / 沉浸 / 氛围」交互盖住或延后。
3. **隐私可删除**：本地昵称身份只存浏览器，后端只用 `user_id` 隔离记忆；提供按 `user_id` 删除后端记忆的脚本（先 dry-run 看影响范围，再 `--confirm`）。现场要能展示「记忆可被删除」。
4. **不把脆弱人群当无保护的卖点**：讲独居老人 / 情绪脆弱人群时，必须同时讲为他们做了什么保护（双触发安全网、输出端复检防复述、承认边界衔接专业），不得只把人群当流量话术。
5. **热线号码须核到官方出处与可拨通性**：现场出现的任何求助热线号码，演前必须核对到官方出处并确认可拨通；如不确定，宁可用「请拨打当地紧急电话 / 联系身边可信任的人」这类更稳妥的表述，不要念一个没核实的号码。

## 7. 现场防翻车清单

### 演前彩排（断网状态下完整跑一遍）

- **断网彩排，确认全链路不白屏**：
  - reflect 走 Mock（无 Key 时自动降级，`get_provider` 返回 `MockProvider`），情绪 / 叙事 / 印记仍完整产出；
  - TTS 朗读在不支持的浏览器降级为「不可用」，不报错、不卡死；
  - 场景图走 24 张本地预设图（不依赖在线生图），断网也能正常显示。
- 三处都确认「降级后仍然有完整体验」，任何一处都不能白屏。

### 演示机切真实混元，跑通一次

- 演示机配置 `LLM_PROVIDER=openai` + 腾讯混元（OpenAI 兼容接口，`OPENAI_BASE_URL` 指向混元 endpoint，`OPENAI_MODEL` 用约定的混元模型），**演前真实跑通一次** reflect，确认能拿到真 LLM 生成的叙事。
- 牢记兜底逻辑：真混元调用失败会**自动降级到 Mock**（`OpenAIProvider` 每个方法都包了 try/except 降级），所以即便现场网络抖动，演示也不会中断——但不要把这当借口，能跑真就跑真。

### 时光机数字必须自洽

- 岛屿成长等级阈值（`island_state_service._growth_level`）：1 级 ≤ 1 条；2 级 2–4 条；3 级 5–9 条；4 级 10–19 条；5 级 ≥ 20 条。
- **要演到第 4 级，预置 seed 必须 ≥ 10 条记忆**；要演到第 5 级需 ≥ 20 条。演前用 `POST /api/identity/seed` 或预置库确认条数。
- **口播说出的条数必须与界面等级一致**：别一边说「岛屿记得你来过十几次」，一边界面显示第 3 级。讲之前先看一眼界面数字，对齐了再说。

## 8. 常见追问 Q&A

**Q1：这算游戏吗？**
算。它有可成长的世界（岛屿随情绪记忆从 1 级长到 5 级、长出灯塔等景物）、有持续的反馈循环（每次倾诉都改变场景与叙事）、有可收集的物件（静默贝壳、心境石、岛屿回应卡留下的物件）、有「离开又回来」的留存设计。它是一款以情绪为核心机制的叙事陪伴游戏，只是它的「胜利条件」不是变强，是被记得。

**Q2：AI 到底做了什么？别又是套壳。**
整条管道里 AI 只做两件事，且都用腾讯混元：读懂你这一刻的情绪、写一段克制的中文叙事与心灵印记。其余全是确定性代码——安全规则、场景状态机、记忆检索。我们故意不让 AI 越界：该确定性的地方绝不交给 AI，这正是套壳产品不会做的事。AI 的角色是「编排管道里的两个关键节点」，不是「包一层 prompt 就叫产品」。

**Q3：怎么保证你的产品不会害到人？**
三道防线，都在代码里：进端双触发安全网（强度阈值 + 关键词黑名单，平静措辞的自伤也能命中）；触发即整段暂停普通叙事、换成求助引导；输出端复检，所有「回看历史」的功能都先剔除高风险记忆，绝不让 AI 复述用户过去的自伤原文。再加上一条产品原则：我们承认自己不是咨询、不是医疗，触发风险时主动把人交回给可信任的人和专业热线。

**Q4：数据隐私怎么处理？**
本地昵称身份只存在浏览器 localStorage，没有账号、密码、JWT、OAuth、没有手机号 / 邮箱等真实身份。后端只拿 `user_id` 和情绪文本，用 `user_id` 隔离记忆。并且记忆可删除——应用内直接提供「删除全部记忆」入口，一键清空该身份在后端的记忆 / 物件 / 私房话 / 向量（`DELETE /api/identity/{user_id}`），另保留命令行脚本做 dry-run 批量清理。隐私对脆弱人群尤其重要，所以我们把「可删除」做成了能当场演示的能力。

**Q5：和市面上那些治愈 App 有什么区别？**
两点本质区别。一是人群：它们默认你愿意表达、想要变好；我们专门接住「说不出、写不动、只想坐一会儿」的那段时间。二是态度：它们常常急着给你方法、给你疗愈承诺；我们刻意去疗效化——轨迹允许有起伏，文案不评判、不预测、不替代专业判断。我们不教你变好，只是记得你来过。

**Q6：为什么不做成全自治的多智能体，那样不是更前沿吗？**
更前沿，但更不负责任。自治智能体之间的协作是不可预测的，而我们的用户是情绪脆弱的人。我们刻意选择了一条可解释、可确定的线性管道：该用 AI 用 AI，该用规则用规则。对这个人群来说，可解释和确定性，比「看起来很 AI」重要得多。

## 九、证据墙：为什么这件事值得做（答辩用）

> 原则：只引用可检索的权威来源；不杜撰具体疗效指标，不用产品内部情绪标签自证。以下以《中国国民心理健康发展报告（2023~2024）》（中科院心理所「心理健康蓝皮书」，联合全国 79 家机构、逾 17 万份问卷，2025 年 4 月发布）为来源。**现场答辩前请再次核对原报告的确切数值。**

- **需求真实且规模化**：报告显示大学生中，**低年级学生的抑郁、焦虑风险高于高年级**——刚进入新环境、最需要低门槛情绪出口的，正是心屿的核心人群。
- **「逃避式刷屏」与情绪风险相关**：短视频日均使用超过 4 小时时，存在抑郁风险的比例约为——青少年 **42.1%**、大学生 **29.3%**、成年职业人群 **34.3%**。心屿提供的是「说出来、被接住」的替代出口，而不是又一个无限下滑的信息流。
- **「被听见」本身就是保护因子**：报告指出，在各类关系中 **同伴 / 同学支持对大学生心理健康的支持作用最大**。心屿把「被一个温柔的存在认真听见、并记得」做成产品内核——这是同伴支持的低门槛延伸。
- **边界自觉**：心屿明确不是心理咨询 / 医疗、不替代专业帮助，定位为「到专业帮助之间那段没人接的路」，并在高风险时第一时间把人导向真实热线与可信任的人。

**一句话**：需求真实（低年级高校生情绪风险高）× 现有出口不健康（逃避式刷屏）× 有效路径明确（被听见 / 同伴支持）——心屿正落在这三者的交点上。

来源：《中国国民心理健康发展报告（2023~2024）》心理健康蓝皮书（中国日报网报道，2025-04-21）。
